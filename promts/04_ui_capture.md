# Этап 04 — UI станции записи (React на PravoUI)

> Проект **«Аудиопротокол»**. Цель — рабочий интерфейс оператора на станции:
> экран записи (устройство, индикаторы уровня, старт/стоп/пауза, явный статус),
> список локальных сессий, настройки (из реестра конфигурации) и диагностика.
> UI **не дублирует бизнес-логику** — только дёргает команды и отображает
> события Rust-ядра (`01`/`02`/`03`).

## Контекст

Дизайн-система — «Правовой кодекс» (PravoUI), переиспользуется из
`ex_system_front/.design-sync` (см. [`../CLAUDE.md`](../CLAUDE.md)). Значения
настроек — реестр [`../docs/configuration.md`](../docs/configuration.md). Код —
`src/` (React 19 + TS). Источник данных — Tauri-команды/события из ядра.

**Это станция захвата, не плеер.** Воспроизведение/транскрипт/протокол —
модуль W2.11 в `ex_system`, здесь их нет. На экране записи — **живые
индикаторы уровня** (не waveform-плеер).

**Границы:** реальный кэш дел для привязки — `05` (здесь поле привязки —
ручной ввод/заглушка под кэш); статусы выгрузки и оффлайн-очередь — `06`
(здесь — отображение статуса по данным манифеста). Компоненты PravoUI:
`Header`, `Card`, `BlockHead`, `Button`, `Tag`, `Field`, `Select`, `Checkbox`,
`EmptyState`, `Skeleton`, `CriticalNotice`, `InfoTip`, `Icon`.

## Результат этапа (Deliverables)

1. **Экран «Запись» (главный).**
   - Выбор устройства ввода (`Select` по `list_audio_devices`), индикатор
     выбранного качества (из настроек).
   - **Живые индикаторы уровня** по каналам (RMS/пик, свой SVG/CSS на токенах,
     питается событием `audio_level`); явный клиппинг.
   - Крупные, недвусмысленные элементы управления: **Старт / Пауза /
     Возобновить / Стоп** (primary oxblood для старта/стопа), хронометраж
     (mono), большой статус сессии (`Recording`/`Paused`/`Recovering`/`Error`)
     по событию `capture_state`.
   - Поле **привязки к делу** (№ дела/ФИО) — пока ручной ввод (в `05`
     подключится кэш дел из `ex_system`).
   - **Индикаторы надёжности:** свободное место, предупреждения (обрыв
     устройства, низкий диск) — через `CriticalNotice`/`Tag` по событиям ядра.
2. **Экран «Сессии».** Список локальных записей из манифеста (`03`): название/
   дата (serif/mono), хронометраж, привязка к делу, статус обработки/выгрузки
   (`Tag`: записывается / готово / выгружается / выгружено / ошибка — тона как
   `STATUS` в оболочке W2.8). Пустое — `EmptyState`.
3. **Экран «Настройки».** Редактирование параметров **строго по реестру**
   [`../docs/configuration.md`](../docs/configuration.md) (устройство, качество,
   сегмент/флаш, путь/шифрование хранилища, ретеншн, сеть/сервер, авторизация,
   зеркало). Валидация; сохранение через `save_settings`.
4. **Экран «Диагностика».** Здоровье устройства, свободное место, последние
   события записи/восстановления (из журнала `03`), статус целостности
   последней сессии, версия/идентичность станции.
5. **Состояния.** Idle (нет записи), Recording, Paused, Recovering (баннер «идёт
   восстановление сессии» + выбор продолжить/закрыть — решение из `02`),
   Device-lost (баннер + авто-резюм), Error, Empty. `Skeleton` при загрузке
   списков.
6. **Доступность.** Клавиатура (старт/стоп — горячие клавиши), `role`/`aria`,
   видимый фокус, недвусмысленная индикация «идёт запись» (нельзя перепутать).
7. **Тесты.** Vitest на ключевые компоненты/состояния: переключение
   `capture_state`, реакция индикаторов уровня на события, баннеры
   предупреждений, рендер списка сессий и пустого состояния, форма настроек
   (валидация, сохранение).

## Шаги

1. Layout-оболочка станции (шапка + навигация по экранам Запись/Сессии/
   Настройки/Диагностика) на компонентах PravoUI.
2. `screens/Record.tsx` + индикаторы уровня + контролы + подписка на события
   ядра (Tauri `listen`).
3. `screens/Sessions.tsx` — список из манифеста.
4. `screens/Settings.tsx` — форма по реестру конфигурации.
5. `screens/Diagnostics.tsx` — здоровье/события/целостность.
6. Тонкий слой `lib/core.ts` — обёртки Tauri-команд/событий (типизированные).
7. Тесты (vitest) + проверка a11y.

## Критерии приёмки (Acceptance)

- Оператор видит устройства, выбирает, видит живой уровень, ведёт запись
  старт/пауза/стоп; статус сессии всегда недвусмыслен.
- Предупреждения (обрыв устройства, низкий диск, восстановление) видны
  баннерами и соответствуют событиям ядра.
- Список сессий отражает манифест; настройки читаются/сохраняются по реестру;
  диагностика показывает реальное состояние.
- UI не содержит дубля бизнес-логики (всё — из ядра); `tsc` чисто, vitest
  зелёный, build чистый. Стиль — PravoUI (токены, без ручного CSS-хардкода).

## Вне объёма этапа

- Кэш дел и логика привязки — `05`. Выгрузка/очередь/прогресс — `06` (здесь —
  статус по манифесту). Упаковка/обновления — `08`.
- Любая обработка аудио/целостность/восстановление — это ядро (`01`–`03`).

## Решено

- **Индикатор уровня:** простые столбики (RMS/пик + клиппинг).
- **Доступ к «Настройкам»:** разделён — оператору доступны рабочие параметры
  (устройство, качество); инфраструктурные/security-настройки (адрес сервера,
  путь/шифрование хранилища, ретеншн, идентичность станции, авторизация) —
  только администратору станции.
