# Этап 08 — Упаковка, дистрибутивы, импортозамещение

> Проект **«Аудиопротокол»**. Цель — собрать устанавливаемые дистрибутивы под
> целевые ОС, обеспечить **оффлайн-установку** (залы бывают изолированы),
> контролируемые обновления, подпись пакетов и подготовку к Реестру
> отечественного ПО. Завершает v1.

## Контекст

Архитектура и матрица ОС — [`../CLAUDE.md`](../CLAUDE.md). CI-каркас заложен в
`00`. Целевые ОС: **Astra Linux SE, РЕД ОС, Windows 10/11** (разработка — на
macOS). Параметры/идентичность — реестр
[`../docs/configuration.md`](../docs/configuration.md).

**Важно по форматам Linux:** Astra Linux — Debian-based (**`.deb`**), РЕД ОС —
RPM-based (**`.rpm`**). Tauri умеет `deb` / `rpm` / `appimage`. ALT — фаза 2.

## Результат этапа (Deliverables)

1. **Дистрибутивы по матрице.**
   - Astra Linux SE — `.deb` (зависимости пакетом: `webkit2gtk`, ALSA/PulseAudio).
   - РЕД ОС — `.rpm` (аналогичные зависимости в RPM-нотации).
   - Windows 10/11 — MSI (или NSIS) + **portable**-вариант.
   - (dev) macOS — `.dmg` для разработки, не целевой деплой.
2. **Оффлайн-установка.** Дистрибутивы устанавливаются без интернета:
   - Windows — **бандл фиксированной версии WebView2 Runtime** (не онлайн-
     бутстраппер), чтобы не тянуть рантайм из сети.
   - Linux — системные зависимости (`webkit2gtk`, аудио) объявлены в пакете;
     задокументировать установку на изолированной Astra/РЕД ОС (локальный
     репозиторий/набор пакетов).
3. **Системные зависимости — аудит и документация.** Полный список рантайм-
   зависимостей по каждой ОС (webkit, аудио-стек) + инструкция установки на
   Astra SE / РЕД ОС, в т.ч. в закрытом контуре.
4. **Контролируемые обновления.** Без публичного авто-апдейта (контур суда):
   обновление **подписанными пакетами** (ручная установка) + опционально
   внутренний (локальный) канал обновлений. Tauri-updater — выключен/настроен
   на внутренний источник, не на интернет.
5. **Подпись.** Подпись пакетов: Windows code signing; Linux — GPG-подпись
   пакетов/репозитория. (ГОСТ-подпись бинаря — фаза 2 `11`.)
6. **Сборка в CI.** Расширить CI (`00`): артефакты под matrix. Реальная сборка
   `.deb`/`.rpm` под Astra/РЕД ОС — на совместимом билдере (контейнер/self-
   hosted раннер); зафиксировать процедуру.
7. **Версионирование и changelog.** Семантическая версия, `tauri.conf.json`,
   журнал изменений; идентификатор сборки/станции.
8. **Подготовка к Реестру отечественного ПО.** Чек-лист: отсутствие
   запрещённых иностранных компонентов в рантайме, лицензии зависимостей,
   правообладатель/идентификатор, эксплуатационная документация. (Подача —
   организационный шаг вне кода.)

## Шаги

1. Настроить таргеты сборки Tauri (`deb`, `rpm`, `nsis`/`msi`,
   `dmg`) и метаданные пакетов (зависимости, идентификатор, иконки).
2. Windows: бандлинг фиксированного WebView2 Runtime для оффлайна.
3. Linux: объявить зависимости в `.deb`/`.rpm`, проверить установку на чистых
   Astra SE / РЕД ОС.
4. Подпись пакетов (Windows/Linux) + ключи в CI/секретах.
5. Внутренний/ручной механизм обновлений (без интернета).
6. CI-джобы упаковки + публикация артефактов; процедура сборки под отеч. ОС.
7. Аудит зависимостей + эксплуатационная документация + чек-лист Реестра.

## Критерии приёмки (Acceptance)

- На **чистой** Astra Linux SE, РЕД ОС и Windows 10/11 дистрибутив
  устанавливается **без интернета**, приложение запускается и ведёт запись.
- Windows-установка не требует онлайн-загрузки рантайма (WebView2 в бандле).
- Пакеты подписаны; обновление выполняется контролируемо (подписанный пакет /
  внутренний канал), без публичного авто-апдейта.
- CI выпускает артефакты под матрицу; процедура сборки под Astra/РЕД ОС
  задокументирована.
- Чек-лист Реестра отечественного ПО заполнен (рантайм без запрещённых
  иностранных компонентов, лицензии, документация).

## Вне объёма этапа

- ГОСТ ЭЦП бинаря/юридически значимая подпись записи — фаза 2 (`11`).
- ALT Linux и иные ОС — фаза 2.
- Фактическая подача в Реестр (организационный процесс) — вне кода.

## Открытые вопросы

- Механизм обновлений: ручные подписанные пакеты (предлагаю дефолт) vs
  внутренний update-сервер — зависит от инфраструктуры суда.
- Наличие билдера/раннера под Astra SE / РЕД ОС для CI-сборки `.deb`/`.rpm`
  (иначе — сборка на стенде по задокументированной процедуре).
- Сертификаты подписи (Windows code signing; GPG для Linux) — кто
  правообладатель/издатель.
