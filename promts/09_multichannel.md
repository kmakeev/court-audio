# Этап 09 — Многоканальная запись по ролям (фаза 2)

> Проект **«Аудиопротокол»**, фаза 2. Цель — расширить захват с одного
> качественного канала (v1) до **N синхронных дорожек, привязанных к ролям**
> (судья, секретарь, стороны, зал). Дорожки по ролям повышают качество
> диаризации/протокола в `ex_system` (W2.11). Первый этап фазы 2 — фундамент
> для меток/ролей в реальном времени (`10`) и видео (`12`).

## Контекст

Архитектура и зафиксированная конфигурация — [`../CLAUDE.md`](../CLAUDE.md)
(таблица v1/фаза 2: «Каналы: 1..N, старт с одного → Многоканал по ролям»).
**Все числовые/именованные параметры — из реестра
[`../docs/configuration.md`](../docs/configuration.md)**; новые параметры этапа
**добавляются в реестр** (раздел «Аудио»), не константами. Код —
`src-tauri/audio/` (+ `recorder/`, `integrity/`, `sync/`). Опирается на ядро
захвата (`01`), надёжность (`02`), манифест/целостность (`03`).

> **Отступление от таблицы v1 согласовывать с заказчиком** (CLAUDE.md). Аппаратная
> развилка (многоканальный интерфейс vs набор USB-микрофонов) и источник единого
> клока — в «Открытых вопросах».

## Результат этапа (Deliverables)

1. **Многоканальный захват.** Несколько источников (каналы многоканального
   интерфейса и/или несколько устройств) → **N дорожек**, каждая пишется как в
   `01` (сегментные WAV PCM, непрерывный сброс). Число и состав — из конфигурации.
2. **Карта «канал ↔ роль».** Назначение каждой дорожке роли (судья · секретарь ·
   обвинение · защита · зал · свидетель…); справочник ролей — настройка.
   Роль попадает в имя дорожки, манифест и метаданные выгрузки.
3. **Синхронизация дорожек.** Семпл-точное выравнивание по общей временной оси;
   при раздельных устройствах — контроль и компенсация дрейфа (drift), фиксация
   рассинхрона в журнале. Единый таймкод сессии для всех дорожек.
4. **Манифест и целостность на дорожку.** Расширить манифест (`03`): `channel_id`
   + `role` на сегмент; SHA-256/хеш-цепочка — per-track; журнал событий на канал.
5. **Мастер-даунмикс (опц.) + пофайловые дорожки.** По настройке — сведённый
   мастер плюс отдельные файлы дорожек; согласовать формат мультитрек-выгрузки с
   `ex_system` (расширение контракта `07`; роли — вход для диаризации W2.11).
6. **UI (`04`).** Экран назначения «устройство/канал → роль», пофайловые метры
   уровня, arm/mute/솔о по дорожке, индикация рассинхрона/обрыва канала.
7. **Надёжность на канал (`02`).** Обрыв одного устройства не роняет остальные
   дорожки (пауза/авто-возобновление только затронутого канала); watchdog и
   контроль диска учитывают суммарный поток N дорожек.
8. **Тесты.** Синхронность дорожек (семпл-выравнивание, wrap сегментов);
   компенсация дрейфа; per-track хеш-цепочка в манифесте; обрыв одного канала не
   влияет на другие; downmix корректен; мультитрек-контракт выгрузки.

## Шаги

1. `audio/devices.rs` — перечисление многоканальных возможностей и нескольких
   устройств; модель «источник → каналы».
2. `audio/capture.rs` — N потоков `cpal` в общий таймкод; кольцевые буферы на
   дорожку; расчёт уровней по каналам.
3. `audio/sync.rs` — единый клок, измерение/компенсация дрейфа между устройствами.
4. `recorder/` — пофайловые сегмент-райтеры с общим таймкодом; ротация.
5. `integrity/` + `store/` — расширение манифеста (`channel_id`/`role`),
   per-track хеш-цепочка, журнал на канал.
6. `sync/` — мультитрек-выгрузка (расширение контракта `07`), роли в метаданных.
7. UI назначения ролей и пофайловых метров (`04`); тесты.

## Критерии приёмки (Acceptance)

- Ведётся синхронная запись N дорожек с корректной привязкой к ролям; дорожки
  семпл-выровнены, дрейф контролируется и фиксируется.
- Обрыв одного устройства/канала не прерывает остальные дорожки и сессию.
- Манифест содержит `channel_id`/`role` и per-track целостность; журнал на канал.
- Мультитрек (+опц. мастер) выгружается по расширенному контракту; роли доходят
  до W2.11.
- Параметры — из конфигурации (реестр расширен); `cargo clippy` чисто; тесты
  зелёные.

## Вне объёма этапа

- Метки/закладки и разметка ролей говорящих **в реальном времени** — этап `10`
  (здесь — статическая карта «канал → роль», не пофразовая разметка).
- Автоматическая диаризация/ASR — остаётся в `ex_system` (W2.11), **не
  дублировать**.
- Видео и A/V-синхрон — этап `12`. ГОСТ-подпись — этап `11`.

## Решено

- v1 фиксирует «1..N, старт с одного качественного канала» (CLAUDE.md); фаза 2 —
  многоканал по ролям. Конкретный состав ролей и число дорожек — настройка,
  утверждается с заказчиком под зал.

## Открытые вопросы

- **Аппаратная база:** один многоканальный АЦП/интерфейс (общий клок, проще
  синхрон) vs набор USB-микрофонов (дрейф, сложнее). Влияет на `audio/sync.rs`.
- **Источник единого клока** и допустимый дрейф между устройствами (порог в
  реестр).
- **Контракт мультитрек-выгрузки** с `ex_system`: отдельные `AudioRecording` на
  дорожку vs одна запись с треками — синхронно править `07`/`06`.
- **Справочник ролей** (фиксированный перечень суда vs произвольные) — от
  заказчика.
