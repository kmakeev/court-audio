# Этап 10.3 — Авторизация оператора через `ex_system`

> Проект **«Аудиопротокол»**, улучшения после v1
> ([`../docs/recommendations.md`](../docs/recommendations.md), п. 3). Цель —
> реализовать зафиксированную политику аутентификации: **оператор обязан
> войти перед стартом записи**, идущую запись истечение токена не прерывает,
> в оффлайн-зале старт идёт по кэшированной сессии. Снимает env-подпорку
> `COURT_AUDIO_OPERATOR_TOKEN`.

## Контекст

Политика уже зафиксирована в реестре
[`../docs/configuration.md`](../docs/configuration.md) (раздел
«Аутентификация»: `auth.operator.required_to_start`,
`auth.operator.cached_session_hours = 24`,
`auth.recording_survives_token_expiry`, `auth.station_identity.required`).
Серверная сторона — JWT-flow `ex_system` (`docs/auth.md` в `ex_system`),
эндпоинты приёма — этап `07`. Текущая подпорка: `sync/mod.rs` —
`OPERATOR_TOKEN_ENV` / `OPERATOR_ID_ENV` / `STATION_ID_ENV`.

> Управление учётками, ролями и парольными политиками — в `ex_system`;
> станция только аутентифицируется и кэширует сессию.

## Результат этапа (Deliverables)

1. **Экран входа оператора.** Логин/пароль → JWT `ex_system`; понятные ошибки
   (нет связи / неверные данные / учётка заблокирована). Фоновое обновление
   токена; выход.
2. **Кэшированная сессия для оффлайна.** Последний успешный вход хранится
   зашифрованно (`store/`) в пределах `auth.operator.cached_session_hours`;
   в оффлайне старт записи разрешён по кэшу с индикацией «оффлайн-режим».
   По истечении кэша — вход обязателен (при `required_to_start = true`).
3. **Непрерывность записи.** Истечение токена/выход оператора **не прерывает**
   идущую запись — блокируется только старт новой сессии. Возврат онлайн —
   тихий рефреш без действий оператора.
4. **Оператор в интерфейсе.** Шапка: ФИО оператора, статус связи с сервером,
   смена оператора между заседаниями без перезапуска приложения.
5. **Идентичность в данных.** `operator_id` из сессии входа — в манифест,
   журнал событий, разметку (`10`) и выгрузку (`06`); `station_id` — учётка
   станции (транспорт выгрузки), как зафиксировано. Env-переменные остаются
   только для тестов/CI и помечаются как тестовая подпорка.
6. **Тесты.** Старт без входа запрещён (по настройке); оффлайн-старт по кэшу в
   окне и отказ за окном; запись переживает истечение токена; `operator_id`
   доезжает до манифеста/выгрузки; смена оператора не трогает идущую сессию.

## Шаги

1. Клиент аутентификации в ядре (`sync/`): login/refresh/logout, хранение
   токенов в памяти, кэш-сессия — зашифрованно через `store/`.
2. IPC: команды входа/выхода/статуса, событие смены состояния auth.
3. Экран входа + оператор и статус связи в шапке (`AppShell`).
4. Гейт старта записи (`recorder/`/IPC) по `auth.operator.required_to_start`
   с учётом кэша; непрерывность идущей записи.
5. Прокладка `operator_id`/`station_id` в манифест/журнал/выгрузку; демонтаж
   env-подпорки из боевых путей.
6. Тесты.

## Критерии приёмки (Acceptance)

- Без входа старт записи невозможен (при включённой настройке); после входа —
  штатная работа, ФИО оператора в шапке.
- В оффлайне старт по кэшированной сессии внутри окна; за окном — отказ с
  понятным сообщением.
- Идущая запись не прерывается истечением токена, выходом и сменой оператора.
- `operator_id`/`station_id` присутствуют в манифесте, журнале и выгрузке;
  боевой код не читает `COURT_AUDIO_OPERATOR_TOKEN`.
- Параметры — из реестра; `cargo clippy` чисто; тесты зелёные.

## Вне объёма этапа

- Управление пользователями/ролями, парольные политики — `ex_system`.
- Разграничение настроек по ролям — этап `10.4` (использует роль из этого
  этапа).
- ГОСТ-подпись — этап `11`.

## Решено

- Вход требуется для **старта** сессии; надёжность записи важнее строгости
  сессии (идущая запись не прерывается) — решение заказчика, зафиксировано
  в `configuration.md`.

## Открытые вопросы

- Формат кэш-сессии (что именно кэшируем: refresh-токен vs локальный «билет»
  с ФИО/ролью) — согласовать с безопасностью `ex_system`.
- Нужна ли **PIN-разблокировка** кэшированной сессии в оффлайне (второй
  фактор на станции) — от заказчика.
- Эндпоинт(ы) auth для станций в закрытом контуре (тот же JWT cookie-flow или
  выделенный token-flow для десктопа) — синхронизировать с `ex_system`.
