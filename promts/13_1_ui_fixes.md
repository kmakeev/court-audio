# Этап 13.1 — Исправления по бэклогу: UI/робастность (R-005, R-007, R-009, R-012)

> Проект **«Аудиопротокол»**, исправления по итогам проверки на станции
> ([`../docs/backlog.md`](../docs/backlog.md)). Пакет мелких, изолированных и
> низкорисковых фиксов UI и робастности — без изменения контрактов API и модели
> гейтов. Каждый пункт уже с найденной причиной в бэклоге; здесь — план правки.

## Контекст

Все четыре пункта — фронт/робастность, не требуют решений заказчика и не задевают
приём в `ex_system`. Параметры (где есть) — реестр
[`../docs/configuration.md`](../docs/configuration.md); «магических чисел» в логике
не вводить (соглашение проекта). Дизайн-система — PravoUI (`src/design/`), CSS-
переменные вручную не задавать.

## Результат этапа (Deliverables)

1. **R-009 — высота кнопки «Активная дорожка».** В блоке «Отметить, кто говорит»
   (`src/screens/Record.tsx:837-846`) кнопка-подсказка `variant="primary"`
   (`height: 44`) выше соседних кнопок ролей `variant="secondary"` (`height: 38`)
   ([`src/design/Button.tsx:19,28`](../src/design/Button.tsx#L19)). Выровнять высоту
   ряда: сохранить акцент активной дорожки, но привести её к высоте кнопок ролей
   (общий размер ряда или размерный проп кнопке). Не «магичить» число — если нужен
   единый размер, ввести его в дизайн-систему.
2. **R-012 — высота `Field` vs `Select`.** В ряду фильтров «Сессии»
   (`src/screens/Sessions.tsx:177-206`) `Field` (`padding: 12px 14px`,
   [`src/design/Field.tsx:42`](../src/design/Field.tsx#L42)) выше `Select`
   (`padding: 10px … 10px`, [`src/design/Select.tsx:177`](../src/design/Select.tsx#L177)).
   Согласовать высоту контролов **в дизайн-системе** (единый вертикальный
   паддинг/высота `Field`/`Select`), чтобы чинилось везде, где они соседствуют, а не
   точечно на экране.
3. **R-007 — ввод запятой в полях-списках.** «Настройки → Разметка» (категории
   закладок, роли говорящих) и «Скорости плеера» пересобирают массив на каждый
   keystroke (`splitList`/`splitNumberList`), из-за чего запятую «не набрать» и
   каретка прыгает в конец (`src/screens/Settings.tsx:138-147,172-176`,
   `src/screens/settings-common.tsx:88`). Редактировать **сырой текст** в локальном
   стейте поля; парсить только на `blur`/сохранении; не реформатировать `value` на
   каждый ввод. Починка — **единым паттерном** для всех полей «через запятую».
4. **R-005 — отсутствующее устройство даёт молчаливую тишину.** Когда сохранённый
   `audio.device` не совпадает ни с одним присутствующим устройством, индикатор/
   захват молча деградируют в тишину (`audio/capture.rs::pick_device`,
   `ipc/selftest_cmds.rs` `selected_present` ловит, но до экрана «Запись» не
   доводит). Довести предупреждение до UI экрана «Запись»/самотеста («выбранный
   микрофон не найден — проверьте настройки»); опц. авто-фолбэк на устройство по
   умолчанию с уведомлением. Задокументировать в
   [`../docs/configuration.md`](../docs/configuration.md)/[`../docs/packaging.md`](../docs/packaging.md):
   при переносе конфига между ОС/машинами чистить `audio.device` и машинно-зависимые
   пути (`storage.root_path`, `reliability.mirror.path`).
5. **Тесты.** Полевой ввод: набор строки с запятой/пробелами в середине сохраняет
   текст и позицию каретки (юнит на буфер поля). Самотест/UI: отсутствующее
   устройство → видимое предупреждение (фикстура «устройства нет»). Кнопки ряда
   ролей и поля ряда фильтров — одной высоты (снапшот/стиль, если применимо).

## Шаги

1. Дизайн-система: единая высота контролов (`Field`/`Select`/размер кнопки ряда) —
   закрывает R-012 и R-009.
2. Поля-списки: отделить буфер редактирования от разобранного массива, парсинг на
   `blur`/сохранении — R-007 (единый паттерн, все поля «через запятую»).
3. Отсутствующее устройство: пробросить `selected_present` в UI «Запись»/самотест,
   опц. авто-фолбэк — R-005; доки о переносимости конфига.
4. Тесты по каждому пункту.

## Критерии приёмки (Acceptance)

- Кнопка «Активная дорожка» и кнопки ролей — одной высоты; поля ряда фильтров
  «Сессии» — одной высоты.
- В полях «через запятую» запятая набирается, каретка не прыгает, пробелы в
  середине не ломаются; парсинг — на выходе из поля/сохранении.
- Отсутствующее устройство даёт явное предупреждение оператору, а не молчаливую
  тишину; доки описывают чистку машинно-зависимых полей при переносе конфига.
- Зелёный CI; новых «магических чисел» в логике нет.
