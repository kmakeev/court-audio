# Этап 00 — Инфраструктура и скаффолд

> Проект **«Аудиопротокол»**. Цель этапа — поднять воспроизводимый каркас
> Tauri-приложения (Rust-ядро + React-UI на PravoUI) и CI под целевые ОС.
> Никакой логики записи здесь нет — только «пустой, но собирающийся и
> запускающийся» скелет с правильной структурой каталогов.

## Контекст

Архитектура и зафиксированные решения — в [`../CLAUDE.md`](../CLAUDE.md).
Стек: Tauri 2.x, Rust (stable), React 19 + TS + Vite. Целевые ОС: Astra
Linux SE, РЕД ОС, Windows 10/11. UI переиспользует дизайн-систему «Правовой
кодекс» из `ex_system_front/.design-sync`.

## Результат этапа (Deliverables)

1. Инициализированный Tauri-проект, который собирается (`cargo build`,
   `npm run build`) и запускается (`npm run tauri dev`) с окном-заглушкой.
2. Структура каталогов `src-tauri/` по [`CLAUDE.md`](../CLAUDE.md) — модули как
   пустые заготовки (stub-файлы с doc-комментарием о назначении):
   `audio/`, `recorder/`, `reliability/`, `store/`, `integrity/`, `sync/`,
   `ipc/`.
3. React-UI: каркас из четырёх экранов-заглушек (Запись · Сессии · Настройки ·
   Диагностика) с роутингом и навигацией; подключённые токены PravoUI (шрифты
   Lora/Inter/JetBrains Mono, палитра paper-cream/ink/oxblood) — чтобы UI сразу
   выглядел в дизайн-системе.
4. Скелет конфигурации приложения (`Settings`) — типизированная схема по
   реестру [`../docs/configuration.md`](../docs/configuration.md) (URL сервера,
   реквизиты JWT, устройство, формат/частота — дефолт **44.1 кГц / 16 бит /
   моно, WAV PCM**, путь хранилища, ретеншн, сеть). Пока только модель + экран
   «Настройки» + персист (без эффекта на запись). Значения — из реестра, не
   константами.
5. CI (GitHub Actions) — матрица сборки **macOS + Ubuntu + Windows** (отеч. ОС
   Astra SE / РЕД ОС — позже, этап `08`, без правок кода); шаги: `cargo check`,
   `cargo fmt --check`, `cargo clippy`, `npm run build`, `npm run tauri build`
   (артефакты под каждую ОС матрицы).
6. `docs/architecture.md` — короткое описание каркаса и точек расширения по
   этапам 01–08.

## Шаги

1. Инициализировать Tauri 2.x (`npm create tauri-app`), шаблон React + TS +
   Vite. Привести имена/идентификаторы (`tauri.conf.json` — productName
   «Аудиопротокол», bundle identifier `ru.<...>.audioprotocol`).
2. Создать модульную структуру `src-tauri/src/{audio,recorder,reliability,
   store,integrity,sync,ipc}/mod.rs` со stub-комментариями назначения и
   `pub` API-заглушками, на которые будут опираться следующие этапы.
3. Подключить базовые крейты в `Cargo.toml` **без использования** (фичами/
   закомментированно, чтобы не тянуть лишнее в сборку 00): ориентиры —
   `cpal`, `hound` (WAV), `rusqlite`, `sha2`, `serde`, `tokio`, `reqwest`.
   Реальное подключение — в профильных этапах.
4. Перенести/подключить дизайн-систему PravoUI: скопировать подход
   `ex_system_front/.design-sync` (config + токены + шрифты), завести
   `.design-sync/` и базовые стили; навык `/design-sync` в `ex_system` —
   ориентир.
5. Собрать React-каркас: layout (тёмная шапка + боковая навигация в духе
   оболочки W2.8), роутинг на четыре экрана-заглушки, переиспользуемые
   компоненты PravoUI (`Header`, `Card`, `BlockHead`, `Button`, `EmptyState`).
6. Реализовать модель `Settings` (Rust + Tauri command `get_settings`/
   `save_settings`, хранение в файле конфигурации Tauri) и экран «Настройки»
   с полями (пока без побочных эффектов на запись).
7. Настроить CI-матрицу и проверки качества (fmt/clippy/tsc).
8. Написать `docs/architecture.md`.

## Критерии приёмки (Acceptance)

- `npm run tauri dev` запускает окно с навигацией по 4 экранам в стиле PravoUI.
- `cargo build` и `npm run build` — без ошибок; `cargo clippy` без warning'ов
  на стабильном профиле; `tsc --noEmit` чисто.
- CI зелёный на Linux и Windows; артефакт сборки под Linux прикладывается.
- Структура `src-tauri/` соответствует [`CLAUDE.md`](../CLAUDE.md); каждый
  модуль-заглушка имеет doc-комментарий о будущей роли.

## Вне объёма этапа

- Любая реальная работа со звуком, файлами записей, выгрузкой, целостностью —
  это этапы 01–07.
- Сборка именно под Astra SE / РЕД ОС в их «боевых» образах — документируется
  здесь, выполняется в этапе `08_packaging`.

## Решено

- Качество по умолчанию — 44.1 кГц / 16 бит / моно, WAV PCM-мастер.
- CI-матрица — macOS + Ubuntu + Windows; Astra SE / РЕД ОС — этап `08`.
- Все параметры — из реестра [`../docs/configuration.md`](../docs/configuration.md).
