# Этап 10.6 — UX-пакет: проверка перед заседанием, поиск, защита записи

> Проект **«Аудиопротокол»**, улучшения после v1
> ([`../docs/recommendations.md`](../docs/recommendations.md), п. 6). Цель —
> закрыть рабочие сценарии секретаря целиком: уверенный старт заседания
> (self-test), быстрый поиск записей, защита идущей записи от бытовых
> «сюрпризов» ОС и внятная обратная связь о сбоях.

## Контекст

База v1 уже есть: горячие клавиши, подтверждения, баннеры надёжности,
восстановление сессий. Этап опирается на плеер (`10.1` — «Прослушать» из
карточки), экспорт (`10.2` — «Экспортировать»), вход оператора (`10.3` —
self-test проверяет авторизацию). Параметры — реестр
[`../docs/configuration.md`](../docs/configuration.md); новые параметры этапа
**добавляются в реестр** (кандидаты: `ux.sound_alerts.enabled`,
`ux.autostart.enabled`, `ux.prevent_sleep_while_recording`).

## Результат этапа (Deliverables)

1. **Проверка перед заседанием (self-test одной кнопкой).** Чек-лист:
   устройство отвечает и уровень сигнала живой, свободного места достаточно
   (пороги `reliability.*`), сервер доступен, вход оператора выполнен
   (`10.3`), незавершённых сессий нет → зелёное «можно начинать»; каждая
   красная позиция — с действием «исправить».
2. **Мастер первого запуска** новой станции: URL сервера, выбор и проверка
   устройства, **тестовая запись с прослушиванием** (`10.1`), проверка
   выгрузки; повторно доступен из «Диагностики».
3. **Экран «Сессии»: поиск и карточка.** Поиск/фильтры (№ дела, дата, статус
   выгрузки), сортировка, пагинация на большие архивы; **карточка сессии**:
   метки/интервалы ролей, события журнала, статус целостности, действия
   «Прослушать» (`10.1`) и «Экспортировать» (`10.2`).
4. **Защита идущей записи от ОС.** Блокировка сна/гибернации на время записи;
   предупреждение (и подтверждение) при закрытии приложения с идущей записью;
   опция автозапуска приложения при входе в систему.
5. **Уведомления о сбоях.** Системные нотификации (обрыв устройства,
   критический диск, ошибка выгрузки) + **опциональный** звуковой сигнал
   (настройка: в зале звук может быть неуместен); дублируют, а не заменяют
   баннеры в приложении.
6. **Мелочи трения.** Отмена последней метки одной клавишей (фиксируется
   событием, как правка — `10`); свободный комментарий к сессии; копирование
   пути/№ дела из карточки; человекочитаемые тексты типовых ошибок ядра
   (словарь код → текст оператору).
7. **Тесты.** Self-test корректно диагностирует каждый пункт (фикстуры
   сбоев); фильтры/поиск сессий; блокировка сна включается/снимается со
   стартом/стопом; закрытие с идущей записью требует подтверждения; отмена
   метки журналируется.

## Шаги

1. Self-test: агрегирующая IPC-команда диагностики + экран чек-листа.
2. Мастер первого запуска (использует self-test как финальный шаг).
3. Сессии: поиск/фильтры/пагинация в манифесте (`store/`) + карточка сессии.
4. Защита записи: prevent-sleep (по ОС), перехват закрытия окна, автозапуск.
5. Нотификации + опц. звук; словарь текстов ошибок.
6. Мелочи трения; тесты.

## Критерии приёмки (Acceptance)

- Секретарь перед заседанием получает однозначное «можно начинать» или список
  конкретных проблем с кнопками исправления.
- Новая станция доводится до рабочего состояния мастером без чтения доков.
- Запись находится по № дела/дате за секунды; из карточки доступны
  прослушивание и экспорт.
- Во время записи станция не засыпает; приложение не закрыть случайно.
- Сбой записи/выгрузки заметен, даже если окно свёрнуто.
- Параметры — из реестра; `cargo clippy` чисто; тесты зелёные.

## Вне объёма этапа

- Сам плеер и экспорт — этапы `10.1`/`10.2` (здесь — точки входа из карточки).
- Мониторинг парка станций/централизованные алерты — фаза «эксплуатация».
- Педаль/аппаратный пульт для меток — открытый вопрос этапа `10`.

## Решено

- Звуковые алерты — строго опциональны (настройка, по умолчанию — уточнить у
  заказчика с учётом этикета зала).
- Отмена метки — журналируемое событие (целостность разметки, как в `10`).

## Открытые вопросы

- Состав чек-листа self-test — согласовать с заказчиком (что суду важно
  проверять перед заседанием).
- Автозапуск: приложение или фоновый сервис-агент — решить при реализации
  (влияет на упаковку `08`).
