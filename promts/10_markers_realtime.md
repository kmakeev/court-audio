# Этап 10 — Метки/закладки и роли говорящих в реальном времени (фаза 2)

> Проект **«Аудиопротокол»**, фаза 2. Цель — дать оператору **разметку по ходу
> заседания**: закладки с таймкодом (начало допроса, реплика, инцидент) и
> отметку **кто говорит** (роль) в реальном времени. Разметка выгружается в
> `ex_system` как подсказки для диаризации/протокола W2.11 — **сам ASR не
> дублируем**.

## Контекст

Архитектура и конфигурация — [`../CLAUDE.md`](../CLAUDE.md) (таблица: «Метки в
зале: Нет (v1) → Закладки + роли говорящих в реальном времени (фаза 2)»).
Параметры — реестр [`../docs/configuration.md`](../docs/configuration.md); новые
параметры этапа **добавляются в реестр**. Код — `src-tauri/` (`ipc/`,
`integrity/`, `store/`, `sync/`) + UI (`04`). Опирается на таймкод/манифест
(`01`–`03`) и, при наличии, на карту ролей из `09`.

> Разметка — **ручная**, оператором. Автоматическое распознавание говорящих —
> в `ex_system`. Отступление от таблицы v1 согласовывать с заказчиком.

## Результат этапа (Deliverables)

1. **Закладки с таймкодом.** Оператор ставит метку в текущий момент записи;
   у метки — категория (из справочника) и опц. свободный комментарий. Семпл-
   точное смещение по общей оси сессии.
2. **Разметка ролей говорящих в реальном времени.** Отметка «сейчас говорит
   <роль>» → интервалы ролей на таймлайне. При многоканале (`09`) роль может
   подставляться из активной дорожки; ручная корректировка возможна.
3. **UI живой разметки (`04`).** Кнопки/горячие клавиши категорий и ролей с
   минимальным трением во время заседания; список меток с правкой/удалением
   до завершения сессии; таймлайн-обзор. Действия не мешают захвату.
4. **Персист + целостность.** Метки и интервалы ролей — в манифест/журнал
   событий (`03`), под хеш-цепочку (tamper-evident): кто/когда поставил.
   Правка метки фиксируется как событие, а не молчаливо перезаписывает.
5. **Выгрузка разметки.** Метки/роли уходят вместе с записью в `ex_system`
   (расширение контракта `07`) как структурированные подсказки для W2.11
   (диаризация/NER/черновик протокола).
6. **Тесты.** Семпл-точность смещения метки; персист через рестарт; хеш-
   цепочка/журнал по меткам; подстановка роли из канала (`09`) и ручная
   корректировка; выгрузка разметки по контракту; разметка не тормозит захват.

## Шаги

1. Модель метки/интервала роли (`store/` + расширение манифеста `03`).
2. `ipc/marker_cmds.rs` — команды «поставить/править/удалить метку», «начать/
   завершить интервал роли»; таймкод из активной сессии.
3. Журнал событий/целостность (`integrity/`): метки в хеш-цепочку, авторство.
4. UI живой разметки (`04`): горячие клавиши, список, таймлайн.
5. `sync/` — включить разметку в выгрузку (расширение `07`).
6. Тесты (точность, персист, целостность, выгрузка).

## Критерии приёмки (Acceptance)

- Оператор ставит закладки и отмечает роли по ходу записи; смещения семпл-точны
  относительно оси сессии.
- Метки/роли переживают рестарт, входят в манифест и хеш-цепочку с авторством;
  правки фиксируются событиями.
- Разметка выгружается в `ex_system` как подсказки W2.11 по расширенному
  контракту.
- Живая разметка не мешает захвату (приоритет записи сохранён).
- Параметры/справочники — из конфигурации (реестр расширен); `cargo clippy`
  чисто; тесты зелёные.

## Вне объёма этапа

- Автоматическая диаризация/распознавание говорящих, ASR, NER, генерация
  протокола — **в `ex_system` (W2.11)**, здесь не дублируется.
- Многоканальный захват и статическая карта «канал → роль» — этап `09`.
- ГОСТ-подпись разметки/записи — этап `11`. Видео-таймлайн — этап `12`.

## Решено

- v1 — без меток в зале (только запись + привязка к делу). Фаза 2 добавляет
  ручную живую разметку; состав категорий — настройка, утверждается с заказчиком.

## Открытые вопросы

- **Таксономия закладок** (фиксированный перечень категорий суда vs
  произвольные ярлыки) — от заказчика; в реестр как справочник.
- **Раскладка горячих клавиш** и эргономика в зале (педаль/пульт?) —
  согласовать.
- **Формат подсказок для W2.11** (схема меток/интервалов ролей) — синхронно
  править контракт `07`/`06` и документ W2.11 в `ex_system`.
