# Этап 10.1 — Встроенный проигрыватель с навигацией по меткам

> Проект **«Аудиопротокол»**, улучшения после v1
> ([`../docs/recommendations.md`](../docs/recommendations.md), п. 1). Цель —
> дать станции **воспроизведение записанных сессий**: сразу после заседания
> проверить запись и найти нужный фрагмент по меткам. Записи зашифрованы
> at-rest, поэтому проигрыватель — часть ядра: дешифровка «на лету», ключ не
> покидает Rust.

## Контекст

Архитектура — [`../CLAUDE.md`](../CLAUDE.md). Параметры — реестр
[`../docs/configuration.md`](../docs/configuration.md); новые параметры этапа
**добавляются в реестр** (кандидаты: `player.seek_step_seconds`,
`player.playback_rates`). Опирается на: сегменты + шифрование at-rest (`03`),
дорожки многоканала (`09`), метки и интервалы ролей (`10`). Код —
`src-tauri/` (новый модуль `player/`, `ipc/`) + UI (`04`).

> Воспроизведение — локальная функция станции. Серверное прослушивание и
> обработка записи — в `ex_system` (W2.11), здесь не дублируются.

## Результат этапа (Deliverables)

1. **Движок воспроизведения в ядре.** Потоковое чтение сегментов с дешифровкой
   на лету (ключ не покидает ядро), склейка по манифесту в непрерывную ось
   сессии, вывод в аудио-устройство. Вариант транспорта (вывод через
   `cpal`/`rodio` vs стрим в webview через кастомный протокол Tauri) — выбрать
   и обосновать в коде/доке.
2. **IPC-команды плеера.** Открыть сессию / play / pause / seek (по времени и
   к метке) / скорость 0.5–2× / громкость / выбор дорожки; событие позиции для
   UI (частота — настройка).
3. **UI «Прослушивание».** Открывается из списка сессий («Прослушать»):
   - таймлайн всей записи с нанесёнными **метками** (категория + комментарий)
     и **интервалами ролей** (`10`); клик по метке — мгновенный переход;
   - список меток рядом с таймлайном — «оглавление» записи;
   - управление: play/pause, перемотка ±N с, скорость, громкость, горячие
     клавиши (согласованные с раскладкой экрана «Запись»);
   - многоканал (`09`): выбор дорожки по роли или сведённый микс;
   - статус целостности сессии виден при прослушивании.
4. **Аудит доступа.** Прослушивание — событие журнала (`integrity.event_log`):
   кто, когда, какую сессию (аудио — ПДн).
5. **Приоритет захвата.** Прослушивание не мешает идущей записи: отдельный
   поток, при конфликте ресурсов запись всегда важнее.
6. **Тесты.** Дешифровка + склейка сегментов (включая усечённый последний);
   точность seek к метке (семпл-точное смещение оси сессии); скорость без
   искажения таймкодов; многоканальный выбор дорожки; параллельная запись не
   деградирует; событие аудита пишется.

## Шаги

1. `player/` в ядре: итератор дешифрованных семплов по манифесту (переиспользуя
   `store/`/`integrity/`), позиционирование по оси сессии.
2. Вывод звука (выбранный транспорт) + управление скоростью/громкостью.
3. `ipc/player_cmds.rs` + события позиции.
4. UI: экран/панель прослушивания, таймлайн с метками/ролями, список-оглавление.
5. Журнал аудита прослушивания.
6. Тесты (склейка, seek, многоканал, приоритет захвата, аудит).

## Критерии приёмки (Acceptance)

- Завершённая сессия воспроизводится на станции без расшифровки файлов на диск;
  ключ не покидает ядро.
- Клик по метке/интервалу роли переводит воспроизведение точно в её смещение.
- Многоканальная сессия: слушается любая дорожка и микс.
- Идущая запись не страдает от параллельного прослушивания.
- Каждое прослушивание — в журнале событий.
- Параметры — из реестра конфигурации; `cargo clippy` чисто; тесты зелёные.

## Вне объёма этапа

- Экспорт записей и автономный плеер для копий — этап `10.2`.
- Редактирование/монтаж аудио — не делаем вовсе (целостность записи).
- ASR/диаризация/протокол — `ex_system` (W2.11).
- Полноценная карточка сессии (события, фильтры) — этап `10.6`; здесь —
  только кнопка «Прослушать» из списка сессий.

## Решено

- Дешифровка только потоковая, в памяти ядра; временных открытых файлов на
  диске не создаём.
- Прослушивание журналируется как доступ к ПДн.

## Открытые вопросы

- **Транспорт звука**: вывод из Rust (`rodio`) vs стрим в webview (`<audio>` +
  кастомный протокол) — сравнить по задержке seek и простоте; решение
  зафиксировать.
- Разрешать ли прослушивание **во время активной записи** той же станцией
  (эргономика зала) — согласовать с заказчиком; технически не блокируем.
