# Этап 10.2 — Экспорт записей: папка, DVD, автономное прослушивание

> Проект **«Аудиопротокол»**, улучшения после v1
> ([`../docs/recommendations.md`](../docs/recommendations.md), п. 2). Цель —
> штатная **выдача копии записи**: участнику процесса, в архив, на носитель.
> Пакет копии самодостаточен: аудио + метки + манифест целостности +
> автономный HTML-плеер, который открывается с диска в любом браузере.

## Контекст

Архитектура — [`../CLAUDE.md`](../CLAUDE.md). Параметры — реестр
[`../docs/configuration.md`](../docs/configuration.md); новые параметры этапа
**добавляются в реестр** (кандидаты: `export.enabled`, `export.default_codec`,
`export.require_confirmation`). Опирается на движок дешифровки/склейки этапа
`10.1`, манифест и хеш-цепочку (`03`), метки/роли (`10`), дорожки (`09`).
Функциональный ориентир — SRS «Фемида» (выдача копий аудиопротокола).

> Экспорт выводит ПДн из зашифрованного хранилища — это **управляемое и
> журналируемое** действие, разрешение задаёт администратор (`10.4`).

## Результат этапа (Deliverables)

1. **Мастер экспорта** из списка/карточки сессии:
   - состав: все дорожки / сведённый микс / одна дорожка (по роли);
   - формат: WAV (без потерь) / FLAC (компактнее, тоже lossless);
   - назначение: папка (в т.ч. USB-носитель) / DVD.
2. **Экспортный пакет** (фиксированная структура, описать в `docs/`):
   - аудиофайлы (дешифрованные, склеенные из сегментов);
   - метки и интервалы ролей: JSON + человекочитаемый список (текст);
   - манифест копии: SHA-256 каждого файла, параметры записи, № дела,
     дата/время, станция/оператор, версия схемы;
   - **автономный HTML-плеер** — одна страница без внешних зависимостей:
     таймлайн, переход по меткам, скорость; работает с диска offline.
3. **Запись на DVD** (data-DVD): через системные средства целевых ОС — IMAPI
   (Windows), `growisofs`/аналог (Astra SE / РЕД ОС); после прожига —
   **верификация** чтением и сверкой SHA-256. Отсутствие привода/утилиты —
   понятная диагностика, экспорт в папку остаётся доступен.
4. **Аудит и контроль.** Каждый экспорт — событие журнала (кто, когда, какая
   сессия, состав, назначение). Настройка администратора: экспорт
   разрешён / запрещён / только с подтверждением.
5. **Тесты.** Побайтовая корректность дешифрованного аудио (хеш против
   оригинала до шифрования); сверка хешей пакета; метки в пакете совпадают с
   манифестом сессии; HTML-плеер открывается и переходит по меткам (смоук через
   тест DOM/фикстуру); отказ экспорта при запрете администратора; журнал.

## Шаги

1. Сборка пакета: дешифровка/склейка (движок `10.1`) → файлы + метки +
   манифест копии; структура пакета — в `docs/`.
2. Автономный HTML-плеер (шаблон в репо, данные подставляются при экспорте).
3. Мастер экспорта в UI (состав/формат/назначение, прогресс, итоговая сводка).
4. DVD: обёртка над системными средствами по ОС + верификация после прожига.
5. Политика разрешения экспорта (настройка администратора) + журнал.
6. Тесты.

## Критерии приёмки (Acceptance)

- Пакет, экспортированный в папку, полностью автономен: аудио слушается в
  HTML-плеере с переходом по меткам на компьютере без установленного ПО.
- Хеши в манифесте копии сходятся; целостность копии проверяема получателем.
- DVD прожигается и проходит верификацию на Windows и целевом Linux; при
  отсутствии привода поведение понятное.
- Экспорт управляется настройкой администратора и всегда журналируется.
- Параметры — из реестра; `cargo clippy` чисто; тесты зелёные.

## Вне объёма этапа

- ГОСТ ЭЦП экспортного пакета — этап `11` (пакет подготовлен: манифест копии —
  будущий объект подписи).
- Audio-CD (CD-DA) — не делаем: только data-DVD/папка (метки и целостность
  в CD-DA не переносятся).
- Отправка копий по сети/почте — вне модуля.

## Решено

- Копия всегда сопровождается манифестом целостности и метками — «голый WAV»
  не выдаём.
- Экспорт — журналируемое действие с политикой разрешения от администратора.

## Открытые вопросы

- **Юридические требования к составу копии** (что обязано быть на носителе для
  выдачи стороне) — от заказчика.
- Нужен ли **сжатый формат для сторон** (mp3/ogg) в дополнение к lossless —
  от заказчика; технически — ещё один кодек в мастере.
- Перечень поддерживаемых утилит прожига на Astra SE / РЕД ОС — уточнить на
  стенде (этап `08` — процедуры стендов).
