# Аудиопротокол

Внешний модуль аудиопротоколирования судебных заседаний — надёжная запись
звука в зале/кабинете с привязкой к делу и выгрузкой в экспертную систему
[`ex_system`](../ex_system/) (обработка — модуль W2.11: ASR/диаризация/NER/
черновик протокола). Функциональный ориентир — SRS «Фемида». В первой версии —
**только аудио**; видео и ГОСТ-ЭЦП — фаза 2.

Модуль рассчитан на работу в **закрытом контуре**: оффлайн-установка,
оффлайн-захват с чанковой догрузкой при появлении связи и — для изолированных
залов — [первый запуск полностью без сети](docs/first_run_offline.md).

## Основные функции

- **Бесперебойная запись.** Сегментная запись WAV (PCM) с непрерывным сбросом
  на диск: при сбое питания теряется максимум незавершённый интервал fsync
  (по умолчанию 1.5 с). Write-ahead журнал сессии — запись восстанавливается
  после сбоя/рестарта; watchdog с авто-рестартом; отключение микрофона не
  роняет сессию (пауза + авто-возобновление); контроль свободного места;
  опциональное зеркало на второй носитель, повторяющее структуру хранилища.
- **Многоканальная запись по ролям.** До N синхронных дорожек (судья,
  секретарь, стороны, зал…) с единым клоком сессии и компенсацией дрейфа;
  роли уходят в `ex_system` как вход диаризации. По умолчанию — один
  качественный канал.
- **Живая разметка.** Закладки с таймкодом и интервалы «сейчас говорит
  <роль>» — вне аудио-потока, семпл-точно, под хеш-цепочкой; подсказки для
  диаризации и протокола W2.11 (ASR на станции не дублируется).
- **Целостность и защита данных.** SHA-256 по сегментам + хеш-цепочка +
  журнал событий (tamper-evident); шифрование at-rest ключом станции
  (AES-256-GCM, ключ из парольной фразы через Argon2id) — аудио-сегменты
  (с этапа 13.7: на диске и зеркале — только `.enc`) и служебные данные
  (кэш сессии, PIN-хеши, кэш дел); настраиваемый
  ретеншн локальных копий (по умолчанию — удаление после подтверждённой
  выгрузки + окно безопасности).
- **Авторизация оператора.** Вход через JWT `ex_system`; оффлайн-старт по
  кэшированной сессии + PIN; для изолированных залов — автономный старт по
  провижиненному PIN без единого онлайн-входа (под флагом, дефолт «выкл.»).
  Идущую запись истечение токена/смена оператора не прерывает. Настройки
  разграничены: оператор/администратор (админ-PIN, гейт на уровне ядра).
- **Привязка к делу.** Кэш докета из `ex_system` для оффлайн-выбора + ручной
  ввод № дела как фолбэк; привязка к `Adjudication`.
- **Выгрузка в `ex_system`.** Возобновляемая чанковая выгрузка (часть = целый
  сегмент), оффлайн-очередь, ретраи с экспоненциальным бэкоффом,
  статус-поллинг; верификация целостности на приёме.
- **Проигрыватель и экспорт.** Встроенный плеер завершённых сессий
  (дешифровка в памяти ядра, навигация по меткам/ролям, выбор дорожки,
  скорости 0.5–2×); экспорт самодостаточной копии (WAV/FLAC + метки +
  манифест целостности + автономный HTML-плеер) в папку/USB или на data-DVD —
  журналируемое действие под политикой администратора.
- **Рабочее место оператора.** Экраны Запись · Сессии · Настройки ·
  Диагностика · Администрирование на PravoUI; self-test перед заседанием;
  мастер первого запуска; «режим зала» с крупным статусом; компакт-окно
  «поверх всех окон»; защита от закрытия окна с идущей записью; адаптивная
  сетка.

## Характеристики

| Аспект | Значение (дефолт; всё настраивается — [`docs/configuration.md`](docs/configuration.md)) |
|---|---|
| Мастер-формат | WAV PCM 44.1 кГц / 16 бит / моно; опц. архивная копия FLAC |
| Сегмент | 30 с — единица файлов, хеширования и выгрузки |
| Потеря при сбое питания | ≤ интервала fsync (1.5 с) |
| Каналы | 1..N дорожек по ролям (многоканал — опционально) |
| Целостность | SHA-256 по сегментам + хеш-цепочка + журнал событий |
| Шифрование | AES-256-GCM at-rest ключом станции (Argon2id из парольной фразы): аудио-сегменты (`storage.encrypt_at_rest`, дефолт вкл.) и служебные кэши/PIN-хеши |
| Секреты | В `settings.json` не хранятся; провижининг через env ([`docs/packaging.md`](docs/packaging.md)) |
| ОС | Astra Linux SE, РЕД ОС, Windows 10/11 (разработка — также macOS) |
| Дистрибутивы | `.deb` / `.rpm` / AppImage / NSIS (с бандлом WebView2 для оффлайн-установки) |
| Стек | Tauri 2.x: Rust-ядро (`cpal`, `rusqlite`) + React 19/TS на дизайн-системе «Правовой кодекс» (PravoUI) |
| Обновления | Контролируемые, без авто-апдейта — ручные подписанные пакеты |

## Первый запуск в полностью оффлайн-режиме

Станция вводится в эксплуатацию без доступа к `ex_system`: оффлайн-установка
дистрибутива, провижининг секретов через переменные окружения
(`COURT_AUDIO_STATION_PASSPHRASE`, PIN-ы, идентификаторы), для изолированного
зала — включение автономного режима и засев операторского профиля, проверка
self-test и пробная запись. Пошаговая инструкция администратора —
[`docs/first_run_offline.md`](docs/first_run_offline.md); модель
аутентификации — [`docs/auth.md`](docs/auth.md).

## Состояние

Текущая версия — **0.2.0**. Реализованы: v1 (этапы 00–08 — захват, надёжность,
хранилище/целостность, UI, привязка к делу, выгрузка, упаковка), многоканал
(09), живая разметка (10), пакет улучшений 10.1–10.6 (плеер, экспорт,
авторизация, роли настроек, адаптивный UI, UX) и исправления бэклога 13.1–13.6
(включая автономный оффлайн-старт). Снимок состояния —
[`docs/status.md`](docs/status.md); журнал изменений —
[`CHANGELOG.md`](CHANGELOG.md). Архитектура и поэтапный план — в
[`CLAUDE.md`](CLAUDE.md) и каталоге [`promts/`](promts/).

## Сборка

```bash
npm install
npm run tauri dev      # запуск в режиме разработки
npm run tauri build    # сборка дистрибутива под текущую ОС
```

обновления и сборка под Astra SE / РЕД ОС — [`docs/packaging.md`](docs/packaging.md).
