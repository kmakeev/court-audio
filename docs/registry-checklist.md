# Чек-лист подготовки к Реестру отечественного ПО (этап 08)

Подготовка модуля **«Аудиопротокол»** к внесению в Реестр российского ПО
(Минцифры). Документ — техническая часть подготовки; **фактическая подача —
организационный процесс вне кода**. Метаданные упаковки — в
[`../src-tauri/tauri.conf.json`](../src-tauri/tauri.conf.json), процедуры сборки
и оффлайн-установки — в [`packaging.md`](packaging.md).

## 1. Рантайм без запрещённых иностранных компонентов

Требование: в работающем модуле нет проприетарных иностранных
рантайм-компонентов, недоступных в РФ; всё либо свободное ПО, либо системный
компонент отечественной ОС.

| Область | Реализация | Статус |
|---|---|---|
| Захват звука | `cpal` → ALSA/PulseAudio (Linux) · WASAPI (Windows) · CoreAudio (macOS-dev) | ✅ Свободно / системное; ОС-специфики хардкодом нет |
| Оболочка UI | Tauri 2.x + системный webview (webkit2gtk на Linux, WebView2 на Windows) | ✅ Свободное ядро; WebView2 — рантайм Microsoft, бандлится оффлайн (см. прим.) |
| Сеть/TLS | `reqwest` + **rustls** (без OpenSSL) | ✅ Свободно; системный OpenSSL для нашего TLS не нужен |
| Хранилище | `rusqlite` (SQLite, `bundled`) | ✅ Public Domain (SQLite), свой бинарь |
| Крипто at-rest | `aes-gcm`, `argon2`, `sha2` (RustCrypto) | ✅ Свободно (MIT/Apache) |
| Экспорт FLAC (`10.2`) | `flacenc` — чистый Rust, без libFLAC-биндингов | ✅ Свободно (Apache-2.0), свой бинарь |
| ASR/NER/протокол | **не в этом модуле** — в `ex_system` (W2.11) | ✅ Вне границы модуля |

> **WebView2 (Windows):** проприетарный рантайм Microsoft. Для целевого
> импортозамещённого деплоя основные ОС — **Astra SE / РЕД ОС**, где оболочка
> работает на свободном `webkit2gtk`. Windows-вариант — для смешанного парка;
> WebView2 бандлится фиксированной версией офлайн (без обращения в сеть). ГОСТ ЭЦП
> и иные требования доверенной среды — фаза 2.

## 2. Целевые отечественные ОС

- **Astra Linux SE** — `.deb`, проверена оффлайн-установка (процедура —
  [`packaging.md`](packaging.md)).
- **РЕД ОС** — `.rpm`, проверена оффлайн-установка.
- Обе — из Реестра отечественного ПО; сборка без зарубежных облачных
  зависимостей.
- ALT Linux — фаза 2.

| Поле | Текущее значение | Действие |

> Плейсхолдеры сознательны: значения не выдумываются. Перед подачей заказчик
> предоставляет правообладателя/идентификатор; правки — в `tauri.conf.json` и
> этой таблице.

## 6. Импортонезависимость сборки

- Сборка и работа на отечественных Linux (Astra SE / РЕД ОС).
- Оффлайн-установка в закрытом контуре без интернета ([`packaging.md`](packaging.md)).
- Без зарубежных облачных сервисов в рантайме и при выгрузке (выгрузка —
  локальный `ex_system` в контуре суда).
- Зависимости ставятся из локальных репозиториев ОС / зеркал.

## Итог

Технические предпосылки внесения в Реестр выполнены при закрытии двух
организационных пунктов: (а) финальные правообладатель/идентификатор,
(б) финализация руководств оператора/администратора (черновики готовы —
вычитка и скриншоты с целевой станции). Подача — вне кода.
