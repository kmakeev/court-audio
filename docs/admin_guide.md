# Руководство администратора

Руководство для ИТ-специалиста суда, отвечающего за развёртывание и
сопровождение станций «Аудиопротокол». Работа оператора —
[`operator_guide.md`](operator_guide.md). Справочные документы: реестр
настроек — [`configuration.md`](configuration.md), упаковка/дистрибутивы —
[`packaging.md`](packaging.md), аутентификация — [`auth.md`](auth.md),
первый запуск без сети — [`first_run_offline.md`](first_run_offline.md).

## 1. Зона ответственности

Администратор: устанавливает и обновляет станцию, провижинит секреты,
настраивает ОС (сон/автозапуск), правит админ-скоуп настроек (сервер,
шифрование, ретеншн, пороги надёжности, политики), тиражирует профиль
станции на залы, разбирает инциденты. Учётки операторов, роли и парольные
политики ведутся в `ex_system`, не на станции.

## 2. Развёртывание станции

0. **Подбор железа** — до закупки сверьтесь с [`hardware.md`](hardware.md):
   профиль нагрузки, формула ёмкости диска, критерии многоканала, ИБП.
1. **Установка дистрибутива** (интернет не требуется):
   - Astra Linux SE / РЕД ОС — `.deb` / `.rpm`;
   - Windows 10/11 — NSIS-инсталлятор (для закрытого контура — вариант с
     фикс-версией WebView2 в бандле).
   Форматы, системные зависимости, подпись пакетов и проверка подлинности —
   [`packaging.md`](packaging.md).
2. **Настройка ОС** — отключить сон/гибернацию на время работы станции,
   настроить автозапуск приложения при входе в систему. Пошагово для каждой
   ОС — [`os_integration.md`](os_integration.md); краткая выжимка
   продублирована на экране «Администрирование».
3. **Провижининг секретов** — см. следующий раздел; выполняется **до**
   первого старта приложения.

## 3. Провижининг секретов (env)

Секреты не хранятся в `settings.json` — передаются переменными окружения в
скрипте запуска / systemd-юните (полные таблицы — [`packaging.md`](packaging.md),
раздел «Провижининг секретов станции»):

| Env | Назначение |
|---|---|
| `COURT_AUDIO_STATION_PASSPHRASE` | Парольная фраза → **ключ станции**: шифрование записей и служебных данных. **Обязательна при каждом старте и должна быть стабильной** — при смене фразы ранее зашифрованное не восстановить |
| `COURT_AUDIO_ADMIN_PIN` | Админ-PIN; на первом запуске сохраняется хешем, дальше env можно снять |
| `COURT_AUDIO_STATION_ID` | Идентификатор станции (учётка транспорта выгрузки) |
| `COURT_AUDIO_OPERATOR_PIN` / `_ID` / `_NAME` / `_ROLE` | Только для изолированных залов с автономным стартом — [`first_run_offline.md`](first_run_offline.md) |

Ключ станции валидируется на старте: при отсутствии фразы приложение громко
диагностирует проблему (лог + пункт self-test «Ключ станции»), а не
деградирует молча. При включённом шифровании (по умолчанию) без ключа запись
не сохранится.

## 4. Первый запуск

На станции со связью: провижининг env → старт → онлайн-вход оператора →
«Мастер первого запуска» / self-test → пробная запись. Для полностью
изолированного зала (автономный старт по PIN, флаг
`auth.operator.autonomous_offline.enabled`) — отдельная пошаговая
инструкция: [`first_run_offline.md`](first_run_offline.md).

## 5. Админ-доступ на станции

Настройки разделены на **оператор-скоуп** (устройство, справочники разметки,
удобства плеера) и **админ-скоуп** (всё остальное). Разграничение действует
на уровне ядра: попытка изменить админ-параметр без разблокировки
отклоняется независимо от UI.

- Разблокировка — **админ-PIN** на экране «Настройки» → «Администрирование
  станции». Работает полностью оффлайн (проверка против локального хеша).
- Если админ-PIN не был задан при развёртывании, админ-изменения недоступны
  (операторские настройки работают всегда).
- **Опасные изменения** — смена адреса сервера, отключение шифрования,
  смягчение ретеншна — требуют явного подтверждения («Подтвердите опасные
  изменения»).
- Каждое сохранение настроек фиксируется в журнале событий: кто, когда,
  что (старое → новое, без секретов).

### Основные группы админ-настроек

Полный реестр с дефолтами — [`configuration.md`](configuration.md). Ключевое:

| Группа | Параметры | На что смотреть |
|---|---|---|
| Сервер и выгрузка | `sync.*` | `server_base_url`; соотношение «размер сегмента ≤ `chunk_size_mb` ≤ серверный лимит» при изменении длины сегмента |
| Шифрование и ретеншн | `storage.*`, `retention.*` | По умолчанию шифрование включено, локальная копия удаляется после подтверждённой выгрузки + 72 ч |
| Надёжность | `recorder.*`, `reliability.*` | Длина сегмента, интервал fsync, пороги диска, зеркало на второй носитель |
| Аудио и многоканал | `audio.*` | Частота/разрядность/каналы; карта дорожек по ролям |
| Политики | `export.policy`, `auth.*`, `admin.*` | Экспорт: разрешён / запрещён / с подтверждением; параметры входа и PIN |
| Интерфейс | `ui.*`, `ux.*` | Режим зала, компакт-оверлей, звуковой сигнал о сбоях |

## 6. Тиражирование: профиль станции

«Администрирование» → **«Экспорт / импорт профиля»**: полная конфигурация
станции в JSON (секретов не содержит) — для тиражирования на залы и
восстановления. Импорт доступен только администратору и журналируется.

При переносе профиля на другую машину **очистите машинно-зависимые поля**
(`audio.device`, `storage.root_path`, `reliability.mirror.path`) — иначе
«молчаливая тишина» несуществующего микрофона или сломанные пути хранилища.
Подробнее — [`configuration.md`](configuration.md), раздел
«Машинно-зависимые настройки».

## 7. Обновления и версии

Авто-обновления отключены осознанно (закрытый контур): обновление — ручная
установка подписанного пакета новой версии поверх текущей; данные и
настройки сохраняются. Версия видна на экране «Диагностика» (блок
«Станция»). Процедура и перспектива внутреннего update-канала —
[`packaging.md`](packaging.md).

## 8. Диагностика и сопровождение

Экран **«Диагностика»**: устройства ввода, свободное место, целостность
последней сессии (сегменты с хешем, хеш-цепочка, финальное звено), последние
события записи, адрес сервера, идентификатор станции и версия. Здесь же —
self-test и мастер первого запуска.

Регулярно/по инциденту:

- **Self-test перед заседанием** — приучите операторов запускать его; «Сбой»
  по ключу станции или диску требует вмешательства администратора.
- **Очередь выгрузки** — на экране «Сессии» не должно копиться «зависших»
  записей при рабочей связи; ретраи идут автоматически с бэкоффом.
- **Свободное место** — пороги предупреждения/критики настраиваются
  (`reliability.disk_*`); ретеншн по умолчанию сам удаляет локальные копии
  после подтверждённой выгрузки.
- **Зеркало** (если включено) — дублирующая копия на втором носителе
  повторяет структуру хранилища и пригодна для независимого восстановления;
  сбой зеркала не прерывает запись, но виден в логе.
- **Журнал событий** сессии (старт/пауза/обрыв/изменения настроек/доступ к
  записи) — первый источник при разборе инцидента; защищён хеш-цепочкой.

### Типовые проблемы

| Симптом | Причина / действие |
|---|---|
| Self-test: «Ключ станции — сбой» | Не задана/изменена `COURT_AUDIO_STATION_PASSPHRASE` — восстановить исходную фразу ([`first_run_offline.md`](first_run_offline.md), «Частые проблемы») |
| «Конфигурация повреждена — обратитесь к администратору» | Битый `settings.json`: гейты закрыты fail-secure; восстановить файл из профиля станции или удалить (применятся безопасные дефолты) |
| Оператор не может войти оффлайн | Истекло окно кэша (24 ч) — нужен онлайн-вход; либо станция без ключа станции |
| Тишина на метрах при выбранном устройстве | Устройство из перенесённого профиля не существует на этой машине — очистить `audio.device` |
| Выгрузка падает до сети («часть больше предела») | Увеличили сегмент, не подняв `sync.chunk_size_mb` и серверный лимит — [`configuration.md`](configuration.md), «Размер части ↔ размер сегмента» |

## 9. Безопасность и ПДн

Аудио заседаний — чувствительные данные. На станции обеспечены: шифрование
at-rest аудио-сегментов и служебных данных (AES-256-GCM ключом станции; с
этапа 13.7 записи лежат на диске и зеркале только шифром `.enc`, ОС-шифрование
тома — дополнительная защита, см. [`hardware.md`](hardware.md)), PIN-ы только
хешами (Argon2id), контроль целостности (SHA-256 + хеш-цепочка), журнал
событий, обязательная авторизация оператора, минимизация локального хранения
(ретеншн). Обязанность администратора — сохранность парольной фразы станции,
актуальность политик (`export.policy`, `retention.*`) и отсутствие реальных
ПДн в тестовых записях при наладке.
