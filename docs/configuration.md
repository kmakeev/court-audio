# Конфигурация «Аудиопротокол» — реестр настроек

> **Принцип:** ни одно значение из этого реестра не «зашивается» константой в
> логику. Все параметры — настройки с дефолтами: на станции это типизированная
> схема `Settings` (Rust) + экран «Настройки» + персист в файле конфигурации
> Tauri; на сервере (`ex_system`) — Django settings / env. В коде — только
> обращение к настройке. Этот файл — единый источник истины; промты этапов
> ссылаются на него.

Колонка «Область»: **С** — станция (Tauri-приложение), **Б** — бэкенд
(`ex_system`/Django).

### Многоканальная запись по ролям (фаза 2 — `promts/09_multichannel.md`)

Расширяет захват с одного качественного канала (v1) до **N синхронных дорожек,
привязанных к ролям**. **Аддитивно и выключено по умолчанию:** при
`audio.multichannel.enabled = false` (или пустом `audio.tracks`) поведение —
ровно v1 (один трек `track_id = 0`).

| Параметр | Обл. | Дефолт | Назначение |
|---|---|---|---|
| `audio.multichannel.enabled` | С | `false` | Включение многоканального захвата (иначе — v1: один трек из `audio.device`/`audio.channels`) |
| `audio.tracks` | С | `[]` | Список дорожек `{ device, channel_index, role, label }`; пусто → один трек из `audio.device` |
| `audio.roles` | С | `["judge","clerk","prosecution","defense","witness","room"]` | Справочник ролей дорожек (настраиваемый; роль трека обязана быть из списка) |
| `audio.sync.clock_master_track` | С | `0` | Индекс дорожки — мастера единого клока сессии |
| `audio.sync.drift_threshold_ms` | С | `50` | Порог рассинхрона между дорожками; превышение фиксируется в журнале |
| `audio.sync.drift_compensate` | С | `true` | Компенсация дрейфа (pad/drop семплов) при раздельных устройствах |
| `audio.master_downmix.enabled` | С | `false` | Опц. сведённый мастер (микс всех дорожек) поверх пофайловых дорожек |

`audio.tracks[*].role` ∈ `audio.roles`. `audio.tracks[*].channel_index` — индекс
канала в интерливнутом потоке выбранного `device` (0-based). Один многоканальный
интерфейс (общий клок) → дрейф ≈ 0, компенсация не срабатывает; тот же код
обслуживает и набор USB-микрофонов (раздельный клок, дрейф компенсируется).
Роли дорожек уходят в `ex_system` как вход диаризации W2.11 (контракт — ниже,
«Выгрузка и сеть»).

## Хранилище и ретеншн (локально на станции)

| Параметр | Обл. | Дефолт | Назначение |
|---|---|---|---|
| `storage.root_path` | С | `<data-dir>/recordings` | Корень локального хранилища |
| `storage.encrypt_at_rest` | С | `true` | Шифрование записей на диске (AES-256-GCM на сегмент) |
| `storage.key_source` | С | `os_keystore` | Источник ключа шифрования at-rest |
| `retention.mode` | С | `until_confirmed_plus_window` | Политика удаления локальной копии |
| `retention.require_integrity_verified` | С | `true` | Удалять только после серверного `integrity_verified=true` |
| `retention.safety_window_hours` | С | `72` | Буфер после подтверждения до удаления |

`retention.mode` ∈ { `until_confirmed_plus_window` (дефолт), `delete_on_confirm`
(минимизация ПДн), `manual` (до ручного удаления) }.

`storage.key_source` ∈ { `os_keystore` (дефолт), `passphrase` }. Секрет в файле
настроек **не хранится**. Для `passphrase` (а также как фолбэк, пока OS-keystore
не интегрирован — этап `08`) ключ — Argon2id из парольной фразы станции,
переданной через env `COURT_AUDIO_STATION_PASSPHRASE`; соль персистится в
`<root_path>/key.salt`. Параметры Argon2id — рекомендованные дефолты крейта
(крипто-константы, не бизнес-логика).

## Выгрузка и сеть

| Параметр | Обл. | Дефолт | Назначение |
|---|---|---|---|
| `sync.server_base_url` | С | — (обяз.) | Базовый URL `ex_system` |
| `sync.chunk_size_mb` | С | `8` | Размер чанка возобновляемой загрузки |
| `sync.parallel_uploads` | С | `1` | Параллельных выгрузок |
| `sync.auto_upload` | С | `true` | Авто-выгрузка по готовности записи (фон) |
| `sync.defer_during_recording` | С | `false` | Откладывать догрузку до конца активной сессии |
| `sync.offline_queue.enabled` | С | `true` | Оффлайн-очередь выгрузки |
| `sync.retry.backoff_base_ms` | С | `2000` | База экспоненциального бэкоффа |
| `sync.retry.backoff_max_ms` | С | `60000` | Потолок бэкоффа |
| `sync.retry.max_attempts` | С | `0` (без лимита) | Лимит попыток (0 = бесконечно, до успеха) |
| `AUDIO_MAX_UPLOAD_PART_MB` | Б | `16` | Серверный лимит размера части |

**Мультитрек-контракт (фаза 2, этап 09).** Одна `AudioRecording` на сессию + N
треков внутри (`tracks[] = { track_id, role, label, final_chain_link,
segments[] }`). Часть адресуется по треку:
`PUT /audio/recordings/{id}/upload/part/{track_id}/{part_index}/`. Роли доходят
до диаризации W2.11. Серверный приём — этап `07` в `ex_system` (правится
синхронно с `promts/06_sync_agent.md`).

**Контракт разметки (фаза 2, этап 10, `manifest_version = 3`).** Манифест несёт
блок `annotations = { chain_final_link, markers[], role_spans[], log[] }`:
`log[]` — append-only лог действий под хеш-цепочкой (для серверной проверки
целостности разметки), `markers[]`/`role_spans[]` — свёрнутое текущее состояние
(подсказки W2.11). Разметка передаётся в `POST /audio/recordings/{id}/upload/init/`
рядом с `tracks`: `{ "tracks": [...], "annotations": {...} }`. Приём и
проброс в W2.11 — этап `07` в `ex_system` (правится синхронно).

## Разметка: метки и роли (фаза 2 — `promts/10_markers_realtime.md`)

Ручная **живая разметка** заседания: закладки с таймкодом (категория + опц.
комментарий) и интервалы «сейчас говорит <роль>». Разметка — подсказки для
диаризации/протокола W2.11 в `ex_system`; ASR/NER здесь не дублируется.
Действия идут **вне аудио-потока** (не мешают захвату), пишутся в write-ahead
журнал сессии (переживают рестарт), под хеш-цепочкой (tamper-evident,
авторство). **Аддитивно:** пустой справочник — просто нет быстрых кнопок.

| Параметр | Обл. | Дефолт | Назначение |
|---|---|---|---|
| `markers.categories` | С | `["Закладка","Инцидент","Перерыв","Прочее"]` | Справочник категорий закладок (человекочитаемые ярлыки; значение уходит в W2.11 как подсказка). Настраивается заказчиком — согласован минимальный набор |

Роли интервалов **переиспользуют справочник `audio.roles`** (нового параметра
нет). В многоканале (этап 09) роль интервала может подставляться из активной
дорожки; ручная корректировка — выбор любой роли из `audio.roles`. Смещения
метки/интервала семпл-точны относительно **общей оси сессии** (единый
`started_at` + частота записи).

## Проигрыватель (улучшение после v1 — `promts/10_1_playback.md`)

Встроенный проигрыватель завершённых сессий: потоковая дешифровка сегментов в
памяти ядра (ключ не покидает Rust), навигация по меткам/интервалам ролей
(`10`), выбор дорожки/микс (`09`). Вывод звука — из Rust (`rodio` поверх уже
используемого `cpal`), plaintext не пересекает границу Tauri IPC.

| Параметр | Обл. | Дефолт | Назначение |
|---|---|---|---|
| `player.seek_step_seconds` | С | `15.0` | Шаг перемотки кнопками ±N с (заседание идёт часами — компромисс между точностью попадания в реплику и числом нажатий) |
| `player.playback_rates` | С | `[0.5, 0.75, 1.0, 1.25, 1.5, 2.0]` | Доступные скорости воспроизведения (диапазон 0.5–2×, стандартный набор аудио/видео-плееров) |
| `player.position_update_hz` | С | `5` | Частота события позиции `player_position` для UI; ниже `audio.level_update_hz` — плейхеду на многочасовой шкале не нужна живость метра реального времени |

## Экспорт (улучшение после v1 — `promts/10_2_export.md`)

Выдача самодостаточной копии записи (участнику процесса/в архив/на носитель):
дешифрованное аудио (WAV/FLAC) + метки/роли + манифест целостности копии +
автономный HTML-плеер, в папку (в т.ч. USB) или на data-DVD. Опирается на
движок дешифровки/склейки этапа `10.1` (побайтовый, не потоковый путь — см.
`docs/export_package.md`). Экспорт выводит ПДн из зашифрованного хранилища —
управляемое настройкой администратора и всегда журналируемое действие
(`EventKind::ExportCreated`, в т.ч. отказ политикой).

| Параметр | Обл. | Дефолт | Назначение |
|---|---|---|---|
| `export.policy` | С | `allowed` | Политика администратора: `allowed` / `forbidden` / `requires_confirmation` |
| `export.default_codec` | С | `wav_pcm` | Предустановленный формат в мастере экспорта (`wav_pcm`/`flac`); оператор может выбрать другой формат прямо в мастере |

Промт `10_2_export.md` называл кандидатами два независимых параметра
(`export.enabled`, `export.require_confirmation`); в реализации они объединены
в один `export.policy` — исключает противоречивую комбинацию (`enabled=false`
при одновременном `require_confirmation=true`) и ложится 1:1 на трёхвариантный
критерий приёмки промта («экспорт разрешён / запрещён / только с
подтверждением»).

## Аутентификация

Решение заказчика: **оператор обязан авторизоваться**. Нюанс надёжности —
вход требуется для **старта** сессии; идущую запись истечение токена/выход не
прерывает; в оффлайн-зале старт идёт по **кэшированной** сессии оператора.

| Параметр | Обл. | Дефолт | Назначение |
|---|---|---|---|
| `auth.station_identity.required` | С/Б | `true` | У станции своя учётка на зал (транспорт выгрузки) |
| `auth.operator.required_to_start` | С/Б | `true` | Оператор обязан войти перед стартом записи |
| `auth.operator.cached_session_hours` | С | `24` | Срок кэш-сессии для оффлайн-старта |
| `auth.operator.offline_pin.required` | С | `true` | Требовать PIN для оффлайн-старта по кэшированной сессии (второй фактор на станции) |
| `auth.operator.offline_pin.min_length` | С | `4` | Минимальная длина PIN (валидация при установке на входе) |
| `auth.recording_survives_token_expiry` | С | `true` | Идущая запись не прерывается при истечении токена |

В модели `AudioRecording` (Б) фиксируются `station_id` **и** обязательный
`operator_id`.

**Клиент аутентификации (этап 10.3).** Вход — JWT-flow `ex_system`
(`docs/auth.md` в `ex_system`): `POST /api/token/` (`{email,password}` →
`{access,refresh}`), `POST /api/token/refresh/` (`{refresh}` → `{access}`),
профиль оператора — `GET /user/` (`id` → числовой `operator_id`, ФИО, роль).
**Кэш оффлайн-сессии** (`store/`, зашифрован AES-256-GCM ключом станции) хранит
`operator_id`/ФИО/роль + **refresh-токен** + хеш PIN (Argon2id) в пределах
`cached_session_hours`; возврат онлайн — тихий refresh без действий оператора.
Секреты (пароль, access/refresh, PIN) в `settings.json` **не хранятся**. Детали —
[`auth.md`](auth.md).
