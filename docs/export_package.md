# Экспортный пакет копии записи (этап 10.2)

> Реализация — `promts/10_2_export.md`. Модуль ядра — `src-tauri/src/export/`.
> Параметры — реестр [`configuration.md`](configuration.md), раздел «Экспорт».

## Назначение

Мастер экспорта (карточка сессии → «Экспортировать») собирает
**самодостаточный пакет** копии записи: для выдачи участнику процесса, сдачи
в архив, записи на носитель. Пакет открывается и слушается **без станции и
без установки ПО** — автономный HTML-плеер работает прямо с диска в любом
современном браузере.

Копия никогда не бывает «голым WAV» (зафиксировано в промте, раздел
«Решено») — манифест целостности и метки/роли сопровождают аудио всегда,
независимо от состава/формата/назначения.

## Структура пакета

```
<дело-или-id-сессии>_<YYYY-MM-DD>/
├── audio/
│   ├── sudya.wav | sudya.flac          # по дорожке (состав «все дорожки»/«одна дорожка»)
│   ├── zaschita.wav | zaschita.flac
│   └── mix.wav | mix.flac              # только при составе «сведённый микс»
├── markers.json                        # AnnotationSnapshot — машиночитаемо
├── markers.txt                         # человекочитаемый список меток/интервалов
├── manifest.json                       # CopyManifest — см. ниже
└── player.html                         # автономный HTML-плеер, офлайн, с диска
```

- **Имя корневой папки** — `<санитизированный № дела или id сессии>_<дата
  старта YYYY-MM-DD>`. Номер дела берётся из `AdjudicationRef.raw_number`
  (если сессия привязана к делу); без привязки — `session_id`.
- **Имена аудиофайлов** — ASCII-слаг роли/метки дорожки
  (`export::package::sanitize_filename`): нижний регистр, латиница/цифры/`-`/`_`,
  кириллица транслитерируется (упрощённо, не по ГОСТ — только для читаемости
  имени файла), пусто после очистки → `track_<id>`. Человекочитаемая
  метка/роль — в `manifest.json`, не в имени файла (ограничения
  Joliet/ISO9660).
- **`markers.txt`** — построчно, отсортировано по смещению:
  `HH:MM:SS  [Категория] комментарий` для меток,
  `HH:MM:SS..HH:MM:SS  Роль: <роль>` для интервалов (`..` без конца — интервал
  ещё открыт на момент выгрузки, штатно не бывает у завершённой сессии).

## `manifest.json` (`CopyManifest`)

Оборачивает существующий манифест записи (`store::export::RecordingManifest`,
источник истины для серверной верификации, этапы 03/09/10) экспортными
полями:

```jsonc
{
  "copy_manifest_version": 1,
  "exported_at_unix_ms": 1_700_000_100_000,
  "exported_by_operator_id": "op-7",
  "composition": { "kind": "all_tracks" },       // | {"kind":"mix"} | {"kind":"track","track_id":0,"role":"judge"}
  "format": "wav_pcm",                            // | "flac"
  "files": [
    { "name": "audio/sudya.wav", "sha256": "…", "size_bytes": 123456, "track_id": 0, "role": "judge" }
  ],
  "recording": { /* RecordingManifest — сегменты, хеш-цепочка, события, annotations.log */ }
}
```

**Две разные величины SHA-256**, обе нужны получателю:
- `recording.tracks[*].segments[*].sha256` — хеш **сегмента до склейки**
  (совпадает с тем, что видел сервер при выгрузке — для сверки с оригиналом).
- `files[*].sha256` — хеш **уже выданного файла** пакета (после склейки и,
  опционально, перекодирования в FLAC) — это хеш, который получатель реально
  может пересчитать над файлом на своём компьютере (`sha256sum` / `shasum -a
  256`) и сравнить с `manifest.json`.

## Целостность и побайтовая точность

- Склейка ([`export::audio`]) — **целочисленный** путь (без f32-нормализации,
  в отличие от потокового плеера этапа 10.1): декодированные семплы дорожки
  до шифрования и после дешифровки+склейки совпадают побайтово.
- Любая ошибка чтения/дешифровки сегмента — **жёсткий отказ** сборки пакета
  (не тихое усечение, как при живом прослушивании) — копия не может незаметно
  потерять хвост записи.
- FLAC-кодирование — из того же целочисленного PCM, без плавающей точки.
  Технический нюанс формата: последний физический фрейм фиксированного
  размера блока может содержать техническую тишину сверх реальной длины
  сигнала — `STREAMINFO.total_samples` хранит настоящую длину, конформные
  декодеры/браузеры проигрывают ровно её.
- «Сведённый микс» — поэлементное целочисленное усреднение дорожек
  (округление к ближайшему), длина = длина кратчайшей дорожки (то же
  упрощение, что у потокового микса плеера 10.1).

## Автономный HTML-плеер

Одна HTML-страница без внешних зависимостей: раскладка и поведение повторяют
встроенный проигрыватель «Прослушивание» (этап 10.1) — таймлайн со
скраббером и засечками меток/интервалов ролей, оглавление с переходом по
клику, транспорт (play/pause/±перемотка/скорость/громкость), горячие
клавиши. Данные (дорожки/метки/роли/снимок `player.seek_step_seconds` и
`player.playback_rates` на момент экспорта) встроены как
`<script type="application/json">` — не через `fetch()`, который блокируется
CORS на `file://` в Chrome; аудио — через один `<audio>` с относительным
`src`, переключаемым при нескольких файлах пакета (медиаэлементы под то же
ограничение не подпадают). Палитра/шрифты — захардкоженные значения токенов
дизайн-системы с системным фолбэком шрифтов (получатель копии не имеет
фирменных шрифтов станции — сознательный компромисс самодостаточности одного
файла). Живая пересборка SHA-256 в плеере не делается — целостность
проверяется отдельно, по `manifest.json`.

## DVD

Прожиг — **data-DVD** (Rock Ridge/Joliet/UDF), не Audio-CD: метки и
целостность в CD-DA не переносятся (вне объёма промта). Обёртка над
системными утилитами: `growisofs` (Linux), IMAPI2 через PowerShell-скрипт
(Windows). После прожига — верификация чтением и сверкой SHA-256 с
`manifest.json`. Отсутствие привода/утилиты — понятная диагностика в
мастере, экспорт в папку остаётся доступен независимо. Метка тома усечена до
32 символов (техническое ограничение Joliet, не параметр реестра).

Прожиг+верификация на реальном приводе — вне автоматических тестов (нет
оптического привода на станции разработки/в CI); построение команды прожига,
поиск утилиты в `PATH` и сверка хешей по «примонтированному» пути
тестируются напрямую.

## Аудит

Каждая попытка экспорта — событие журнала `EventKind::ExportCreated`
(`detail.result` = `"ok"`/`"denied"`), включая отказ политикой
администратора (`Settings.export.policy`). Прожиг DVD журналируется отдельным
событием (`detail.destination.kind = "dvd"`, `detail.destination.verified`).
