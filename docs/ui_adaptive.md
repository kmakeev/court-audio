# Дизайн и адаптивность — ручной чек-лист (этап 10.5)

Проверка деливераблов [`promts/10_5_ui_adaptive.md`](../promts/10_5_ui_adaptive.md),
которые не покрываются автотестами (реальная раскладка, нативный трей, второе
окно). Автотесты (адаптивные смоук-тесты, деривация статуса трея, режим зала и
оверлей через фикстуры) — в `src/**/*.test.tsx` и `cargo test`.

Параметры режима зала/оверлея — реестр [`configuration.md`](configuration.md)
(`ui.hall_mode.enabled`, `ui.compact_overlay.enabled`), правятся администратором
на экране «Администрирование» → карта «Интерфейс».

## Запуск

```
npm run tauri dev
```

## Матрица «монитор / масштаб / ввод»

Прогнать каждый экран (Запись · Сессии · Настройки · Администрирование ·
Диагностика) в трёх режимах. Критерий: **нет горизонтального скролла и
обрезаний**, все контролы доступны.

| # | Условие | Что проверить |
|---|---|---|
| 1 | Окно ~1280×720, масштаб ОС 100 % | Сайдбар 232 px с подписями; сетки карточек в 2–3 колонки; формы не сжаты |
| 2 | Окно сужается < 900 px | Боковая навигация сворачивается в **иконочный рельс** (подписи скрыты, смысл — по tooltip/aria); контент занимает освободившееся место |
| 3 | Окно < 640 px (или узкий моноблок) | Рельс уходит в **бургер** (кнопка в шапке); навигация выезжает оверлеем, закрывается по выбору пункта, клику по затемнению и Esc |
| 4 | Масштаб ОС 150 % (эффективно ~853×480 при 1280×720) | Все экраны работоспособны без обрезаний; текст читаем |
| 5 | Тач-моноблок (или эмуляция тача) | Цели нажатия ≥ 44 px (кнопки/пункты навигации); бургер и пункты попадаемы пальцем |

## Глобальный индикатор записи

- [ ] Начать запись на экране «Запись».
- [ ] Перейти на «Сессии»/«Настройки»/«Диагностику» — в **шапке** виден статус
      («Идёт запись») и хронометр; клик по индикатору возвращает на «Запись».
- [ ] Поставить на паузу — индикатор становится «Пауза» (золотой тон).
- [ ] Остановить — индикатор в шапке пропадает.

## Системный трей

- [ ] В трее ОС появилась иконка приложения; tooltip — «Аудиопротокол · Готов к
      записи».
- [ ] Начать запись → tooltip меняется на «Аудиопротокол · ● Идёт запись»;
      пауза → «❚❚ Пауза»; стоп → «Запись завершена».
- [ ] Меню трея: «Показать окно» разворачивает/фокусирует главное окно; «Выход»
      завершает приложение.

## Режим зала («большой статус»)

- [ ] Кнопка «Режим зала» в шапке (видна при `ui.hall_mode.enabled`).
- [ ] Открывает полноэкранную панель: крупный хронометр, состояние, крупный метр
      уровня — читается с нескольких метров.
- [ ] Фокус переходит на кнопку «Свернуть»; Esc и кнопка закрывают панель.
- [ ] При `ui.hall_mode.enabled = false` (Администрирование → Интерфейс) кнопка
      не показывается.

## Компакт-окно «поверх всех окон»

- [ ] Включить `ui.compact_overlay.enabled` (Администрирование → Интерфейс,
      после разблокировки админ-PIN), сохранить.
- [ ] Кнопка «Окно поверх» в шапке открывает маленькое окно со статусом+хронометром.
- [ ] Окно остаётся **поверх** других программ (переключиться в другое ПО).
- [ ] Статус в окне синхронен с записью (старт/пауза/стоп).
- [ ] Кнопка «✕» в окне закрывает его; повторный «Окно поверх» открывает заново.
- [ ] При выключенной опции кнопки нет, а прямой вызов команды ядром отклоняется.

### Windows: белая рамка и отзывчивость (R-006, этап 13.6)

Дефект R-006 воспроизводился **только на Windows** (WebView2); финальная приёмка —
на Windows-станции (разработка идёт на macOS, где дефект не проявляется). Чистая
логика окна покрыта юнит-тестом (`overlay_spec` в `ipc/ui_cmds.rs`, предикат
маршрута в `lib/window-route.ts`); нативный рендер/фокус — только руками:

- [ ] **Windows:** «Окно поверх» открывается **без белой рамки/вспышки** — фон
      окна сразу тёмный (тема `--dark`), даже до полной отрисовки статуса.
- [ ] **Windows:** после открытия оверлея кнопки **старт/стоп** в основном окне
      реагируют сразу, **без предварительного клика** по основному окну (оверлей
      не забирает фокус — `focused(false)`).
- [ ] Оверлей всё равно остаётся **поверх** других программ (`always_on_top`).
- [ ] **macOS/Linux:** регресса нет — окно открывается и ведёт себя как прежде.

### Управление воспроизведением из окна

Окно двухрежимное: при активном воспроизведении оно превращается в мини-плеер.

- [ ] Открыть сессию в проигрывателе («Сессии» → «Прослушать»), начать
      воспроизведение, открыть «Окно поверх».
- [ ] Окно показывает **позицию/длительность** и транспорт (◄ · play/pause · ►),
      а не статус записи.
- [ ] Позиция в окне идёт синхронно с проигрывателем; пауза/играть в окне и на
      экране синхронны (общий движок ядра).
- [ ] Переключиться в другое приложение — звук играет в фоне, окно остаётся
      сверху; в фокусе окна **Пробел** ставит паузу/играет, **←/→** — перемотка
      на `player.seek_step_seconds`.
- [ ] Если параллельно идёт запись — внизу окна виден компактный индикатор
      «● Идёт запись ЧЧ:ММ:СС» (запись не теряется из виду).
- [ ] Закрыть проигрыватель (уйти с экрана «Прослушивание») — окно возвращается
      к статусу записи (или к «Готов к записи»).

## Доступность

- [ ] Полный обход интерфейса с клавиатуры (Tab/Shift+Tab), включая навигацию,
      формы, диалоги подтверждения и режим зала.
- [ ] Видимая фокус-обводка на всех интерактивных элементах (оксблад-контур).
- [ ] При системной настройке «уменьшить движение» (`prefers-reduced-motion`)
      анимация метров/скелетонов/выезда навигации отключена.
- [ ] Контраст текста и статусов в норме (тёмная шапка / бумажный контент).
