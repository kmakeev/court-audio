# Интеграция с ОС: сон, гибернация, автозапуск (этап 10.6)

> Часть UX-пакета этапа `10.6` ([`../promts/10_6_ux_polish.md`](../promts/10_6_ux_polish.md),
> deliverable 4). **Блокировка сна/гибернации** во время записи и **автозапуск**
> приложения при входе в систему реализуются **средствами ОС при развёртывании**
> (этап `08_packaging`), а не тумблерами/кодом приложения — чтобы не плодить
> неработающие настройки-заглушки. Этот документ — платформенные инструкции для
> администратора станции и ручной чек-лист проверки. Экран «Администрирование»
> дублирует краткую выжимку.

Что реализовано **в приложении** (без плагинов ОС):

- **Подтверждение при закрытии окна с идущей записью** — core-API
  `onCloseRequested` (`src/shell/useCloseGuard.ts`): при активной записи закрытие
  перехватывается и требует подтверждения; уже сохранённые сегменты не теряются.
- **Опциональный звуковой сигнал о сбоях** — `ux.sound_alerts.enabled` (реестр
  [`configuration.md`](configuration.md)), Web Audio в webview (`src/lib/alerts.ts`).
  Дублирует, не заменяет баннеры/трей. По умолчанию выключен (этикет зала).

Что настраивается **средствами ОС** (ниже).

## 1. Отключение сна/гибернации во время работы станции

Станция ведёт непрерывную запись заседания часами — уход ОС в сон недопустим.

### Astra Linux SE / РЕД ОС (Linux)

- Замаскировать цели сна systemd (общесистемно):

  ```bash
  sudo systemctl mask sleep.target suspend.target hibernate.target hybrid-sleep.target
  ```

- В параметрах рабочего стола (Fly / GNOME / KDE) выставить «Гашение экрана» и
  «Переход в сон» в **«Никогда»** для сеанса оператора.
- Для несменяемой станции-моноблока — при необходимости отключить
  `HandleLidSwitch=ignore` в `/etc/systemd/logind.conf`.

### Windows 10/11

- «Панель управления» → «Электропитание» → схема **«Высокая
  производительность»**; «Переход в спящий режим» = **«Никогда»** (от сети).
- Либо командами:

  ```bat
  powercfg /change standby-timeout-ac 0
  powercfg /change hibernate-timeout-ac 0
  powercfg /change monitor-timeout-ac 0
  ```

## 2. Автозапуск приложения при входе в систему

Оператору не нужно вручную запускать станцию — приложение поднимается при входе.

### Astra Linux SE / РЕД ОС (Linux)

- Пользовательский автозапуск: положить `.desktop`-файл в
  `~/.config/autostart/court-audio.desktop` (или системный
  `/etc/xdg/autostart/`):

  ```ini
  [Desktop Entry]
  Type=Application
  Name=Аудиопротокол
  Exec=/usr/bin/court-audio
  X-GNOME-Autostart-enabled=true
  ```

- Для киоск-станции — пользовательский `systemd`-юнит (`systemctl --user enable
  court-audio.service`) с `Restart=on-failure`.

### Windows 10/11

- Ярлык приложения в папке автозагрузки: `Win+R` → `shell:startup` → поместить
  туда ярлык на исполняемый файл станции.
- Либо «Планировщик заданий» → задача с триггером **«При входе в систему»**
  (устойчивее к обновлениям, можно задать «перезапускать при сбое»).

> **Открытый вопрос промта (решается при упаковке, этап 08):** автозапуск самого
> приложения или отдельного фонового сервис-агента. По умолчанию — автозапуск
> приложения (простой сценарий одного оператора на станцию).

## 3. Ручной чек-лист проверки (native)

Автотесты покрывают чистую логику (`useCloseGuard`, `shouldBeep`, self-test); сами
ОС-эффекты проверяются вручную на целевой станции:

- [ ] Во время идущей записи станция **не уходит** в сон/гашение экрана в течение
      длительного заседания (проверить ≥ таймаута сна по умолчанию до настройки).
- [ ] Попытка закрыть окно приложения во время записи **показывает подтверждение**;
      «Остаться» — окно не закрывается, запись продолжается.
- [ ] После перезагрузки/входа в систему приложение **поднимается автоматически**.
- [ ] При `ux.sound_alerts.enabled = true` обрыв устройства / критический диск /
      ошибка выгрузки **сопровождаются звуковым сигналом**; при `false` — тишина,
      баннеры на месте.

## 4. Перспектива (вне объёма 10.6)

- **Фоновые системные нотификации** при свёрнутом окне (обрыв устройства и т.п.)
  требуют плагина ОС-уведомлений (`tauri-plugin-notification`) — вынесено за рамки
  этапа (решение: без новых плагинов). Сейчас внимание к сбою обеспечивают
  in-app баннеры, системный трей (этап 10.5) и опциональный звук.
