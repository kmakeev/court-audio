# Журнал изменений — «Аудиопротокол»

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
версионирование — [семантическое](https://semver.org/lang/ru/).

## [Не выпущено]

Исправления по бэклогу проверки на станции ([`docs/backlog.md`](docs/backlog.md)),
этапы 13.1–13.6.

### Добавлено

- **13.6 · Автономный оффлайн-старт (B-001, согласовано с заказчиком).** Первый
  запуск изолированного зала **без единого онлайн-входа**: разблокировка по
  провижиненному PIN, идентичность из зашифрованного профиля зала
  (`operator_profile.enc`), под флагом `auth.operator.autonomous_offline.enabled`
  (дефолт «выкл.»). Записи помечаются признаком `autonomous_offline` (контракт
  `07`). Руководство — [`docs/first_run_offline.md`](docs/first_run_offline.md).
- **13.2 · Превью многоканала (R-008).** Мониторинг дорожек до старта записи.
- **13.3 · Структура зеркала (R-010).** Зеркало повторяет дерево основного
  хранилища (сессия/дорожка/сегменты + метаданные) — реконсилируемо независимо.

### Исправлено

- **13.1 · UI (R-005/007/009/012).** Правки экранов и дизайн-системы.
- **13.4 · Экспорт (R-011).** HTML-плеер не зависает на метке выступающего.
- **13.5 · Безопасность (R-003/R-004).** Fail-secure гейтов при повреждённом
  `settings.json` (порча не размыкает авторизацию/админ-PIN); ключ станции
  обязателен и валидируется на старте — сбои шифрования не проглатываются.
- **13.6 · Windows overlay (R-006).** Компакт-окно «поверх всех окон» без белой
  рамки и подвисания старта/стопа (финальная приёмка — на Windows-станции).

## [0.2.0] — 2026-07-03

Пакет улучшений после v1 (фаза-2 этапы 09–10 и рекомендации
[`docs/recommendations.md`](docs/recommendations.md), этапы 10.1–10.6).

### Добавлено

- **09 · Многоканальная запись по ролям.** N синхронных дорожек с единым клоком
  и компенсацией дрейфа; аддитивно, по умолчанию выключено (поведение v1).
- **10 · Живая разметка.** Закладки + интервалы «сейчас говорит <роль>» вне
  аудио-потока, под хеш-цепочкой; подсказки диаризации W2.11.
- **10.1 · Проигрыватель.** Встроенный плеер сессий: потоковая дешифровка в
  памяти ядра, навигация по меткам/ролям, скорости 0.5–2×, аудит доступа.
- **10.2 · Экспорт.** Самодостаточная копия (WAV/FLAC + метки + манифест +
  автономный HTML-плеер) в папку/USB/DVD; политика администратора, журнал.
- **10.3 · Авторизация оператора.** JWT-вход через `ex_system`, кэш
  оффлайн-сессии (шифрованный), оффлайн-старт по PIN, идентичность в
  манифесте/журнале/выгрузке; идущую запись истечение токена не прерывает.
- **10.4 · Разграничение доступа.** Оператор/администратор: админ-скоуп
  настроек под админ-PIN (гейт ядра), журнал изменений, экспорт/импорт профиля
  станции.
- **10.5 · Адаптивный UI.** Сворачивание навигации, глобальный индикатор записи
  (шапка + трей), «режим зала», компакт-окно поверх всех окон.
- **10.6 · UX-пакет.** Self-test перед заседанием, мастер первого запуска,
  поиск/карточка сессии, подтверждение закрытия при записи, опц. звуковой
  сигнал о сбоях; рекомендации по ОС — [`docs/os_integration.md`](docs/os_integration.md).

## [0.1.0] — 2026-07 (v1)

Первая версия модуля захвата: только аудио, оффлайн-захват с чанковой догрузкой
в `ex_system`. Итог этапов 00–08.

### Добавлено

- **00 · Инфраструктура.** Скаффолд Tauri 2 (Rust-ядро + React 19/PravoUI),
  схема `Settings` (зеркало `docs/configuration.md`), CI-матрица
  macOS/Ubuntu/Windows.
- **01 · Ядро захвата.** `cpal`: устройства, поток, кольцевой буфер, сегментный
  WAV-райтер (PCM), индикаторы уровня.
- **02 · Запись и надёжность.** Автомат сессии, непрерывный сброс + ротация,
  восстановление после сбоя/питания, watchdog, обрыв устройства, контроль диска.
- **03 · Хранилище и целостность.** SQLite-манифест, шифрование at-rest
  (AES-256-GCM), SHA-256 + хеш-цепочка сегментов, журнал событий, ретеншн.
- **04 · UI записи.** Экраны Запись · Сессии · Настройки · Диагностика на
  PravoUI; метры уровня, старт/стоп/пауза, индикатор сохранения.
- **05 · Привязка к делу (оффлайн).** Кэш дел из `ex_system` + ручной ввод №
  как фолбэк; привязка записи к `Adjudication`.
- **06 · Агент выгрузки.** Возобновляемая чанковая выгрузка, оффлайн-очередь,
  ретраи/бэкофф, статус-поллинг.
- **07 · Приём (в `ex_system`).** Django-приём записи, верификация целостности,
  передача в пайплайн W2.11 — контракт API выгрузки.
- **08 · Упаковка и импортозамещение.** Дистрибутивы по матрице
  (`.deb`/`.rpm`/NSIS·MSI/`.dmg`), объявленные системные зависимости,
  **оффлайн-установка** (бандл фикс-версии WebView2 для Windows), подпись
  пакетов (Windows code signing / Linux GPG), контролируемые обновления
  **без авто-апдейта** (ручные подписанные пакеты), CI-артефакты под матрицу и
  процедура сборки под Astra SE / РЕД ОС, чек-лист Реестра отечественного ПО.
  macOS-бандл: `Info.plist` (`NSMicrophoneUsageDescription`) и `entitlements.plist`
  (`audio-input`) для доступа к микрофону; пошаговая сборка под Ubuntu/Windows и
  сводка разрешений микрофона по ОС — `docs/packaging.md`.

### Отложено (фаза 2)

Видео (A/V-синхрон), многоканал по ролям, закладки/роли в реальном времени,
ГОСТ ЭЦП (КриптоПро), онлайн-стриминг, ALT Linux, внутренний update-канал.
